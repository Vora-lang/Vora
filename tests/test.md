# 1. 纯循环压力测试（最基础、最公平）
## 题目
循环 10 亿次（1,000,000,000），每次只做简单加法：
```
sum = 0
for i from 1 to 10亿:
    sum += 1
输出 sum
```

## 测试点
- 语言**循环速度**
- 解释型 / 编译型差距极大
- 整数运算效率

## 评分
- C/C++：< 0.5 秒
- Go/Rust：1～2 秒
- Java：2～4 秒
- JavaScript：10～30 秒
- Python：**60 秒以上**（甚至几分钟）

# 2. 斐波那契数列递归 + 循环双测（栈 + 计算性能）
## 题目
计算斐波那契第 40 项：`F(40)`
用**递归**和**循环**各写一次。

## 测试点
- 递归函数调用开销
- 栈深度
- 循环效率

## 评分
- 递归：C++ 瞬间出，Python 要等很久
- 循环：所有语言都快，但**速度差距依然明显**

# 3. 大数组读写测试（内存性能）
## 题目
创建一个包含 **1 亿个整数** 的数组
- 全部赋值为 1
- 全部求和

```
arr = [1] * 100000000
sum(arr)
```

## 测试点
- 内存分配速度
- 数组访问效率
- GC（垃圾回收）压力

## 评分
- C++/Rust：秒级
- Java/Go：稍慢
- Python：**极慢，甚至内存爆炸**

---

# 4. 素数筛选（经典 CPU 密集型）
## 题目
用埃氏筛法求 **2 到 1 亿（100,000,000）之间有多少个素数**

## 测试点
- 位运算 / 数组操作效率
- 纯数学计算压力
- 最能体现语言**真实算力**

## 评分
这是最公平的性能题，所有语言算法完全一样，只比速度。
