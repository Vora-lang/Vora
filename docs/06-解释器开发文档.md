# 解释器（Interpreter）开发文档

## 概述

解释器是 Vora 的执行引擎，采用 **树遍历（Tree-Walking）** 方案直接遍历 AST 节点求值。

- **相关文件**：`src/interpreter/interpreter.h`、`src/interpreter/interpreter.cpp`
- **命名空间**：`vora`
- **行数**：`.h` ~160 行，`.cpp` ~1150 行

## 核心设计

### 执行模型

```
Program
 └── for each Stmt → execute(stmt)
                       │
                       ├── LetStmt → defineBinding(name, evaluate(initializer)) [支持 :int/:float 标注]
                       ├── ExprStmt → evaluate(expression)
                       ├── IfStmt → if isTruthy(evaluate(condition)) ...
                       ├── WhileStmt → while isTruthy(evaluate(condition)) ... [支持 break/continue]
                       ├── ForStmt → for element in evaluate(iterable) ... [数组/字符串/range]
                       ├── FuncStmt → 创建 VoraFunction，绑定到环境
                       ├── ObjStmt → 创建 ObjectClass + ObjectConstructorCallable
                       ├── ReturnStmt → throw ReturnSignal(evaluate(value))
                       ├── BreakStmt → throw BreakSignal{}
                       ├── ContinueStmt → throw ContinueSignal{}
                       └── BlockStmt → pushScope() → 执行 → popScope()
```

### 内建函数注册

解释器构造时注册六个内建函数：

| 函数 | 参数个数 | 说明 |
|------|---------|------|
| `print(...)` | -1（变参） | 输出值，多个参数以空格分隔 |
| `clock()` | 0 | 返回当前时间戳（秒，毫秒精度） |
| `input(prompt?)` | -1（变参） | 从 stdin 读取一行文本；可选 prompt 参数 |
| `int(value)` | 1 | 转为整数（截断小数）；接受 number/string/bool |
| `float(value)` | 1 | 转为浮点数；接受 number/string/bool |
| `range(...)` | -1（变参） | `range(stop)` / `range(start, stop)` / `range(start, stop, step)` |

### 作用域管理

解释器使用 **作用域栈**（`deque<Environment>`）管理词法作用域：

```cpp
std::shared_ptr<Environment> globals;    // 全局作用域
Environment environment;                  // 当前作用域
std::deque<Environment> scopeStack;      // 作用域栈
```

**`pushScope()`**：
```
scopeStack: [env_A, env_B]
                ↓ pushScope()
scopeStack: [env_A, env_B, env_C]   ← environment 移入栈
environment: new_env(enclosing=env_C)  ← 新的当前作用域
```

**`popScope()`**：
```
scopeStack: [env_A, env_B, env_C]
                ↓ popScope()
scopeStack: [env_A, env_B]
environment: env_C (从栈顶恢复)
```

### ReturnSignal / BreakSignal / ContinueSignal — 非局部控制流

利用 C++ 异常实现非局部跳转：

```cpp
namespace {
    struct ReturnSignal {
        Value value;
    };
    struct BreakSignal {};
    struct ContinueSignal {};
}
```

- `ReturnSignal`：`execute()` 执行 `ReturnStmt` 时抛出；`callFunction()` 的外层 `try-catch` 捕获并返回 `signal.value`；顶层代码中的 `return` 在 `interpret()` 中被捕获并报错
- `BreakSignal`：`BreakStmt` 抛出，在 `WhileStmt` / `ForStmt` 的循环体中捕获，跳出循环
- `ContinueSignal`：`ContinueStmt` 抛出，在 `WhileStmt` / `ForStmt` 的循环体中捕获，跳至下一次迭代

## evaluate() — 表达式求值

使用 `dynamic_cast` 对每种 `Expr` 子类分发：

### AssignmentExpr

```
evaluate(value) → 环境.assign(name, value) → 返回 value
```

### CallExpr

```
evaluate(callee) → evaluate 各参数 → invoke(callee, arguments, paren)
```

`invoke()` 中：
1. 检查 callee 是否为 `shared_ptr<Callable>`
2. 通过 `dynamic_pointer_cast` 区分 `NativeFunction` 和 `VoraFunction`
3. 校验参数个数（arity check）
4. 调用 `callable->call()`

### VariableExpr

```
环境.get(name, token) → 返回值
```

如果变量未定义，`Environment::get()` 沿作用域链向上查找，找不到则抛出 `RuntimeError`。

### ArrayExpr

```
evaluate 每个元素 → 构造 shared_ptr<Array> → 返回
```

### IndexExpr

```
evaluate(array) + evaluate(index)
→ 检查 array 是 shared_ptr<Array>
→ 检查 index 是非负整数
→ 返回 elements[index]
```

### LiteralExpr

- 如果是字符串，调用 `interpolateString()` 处理插值
- 其他类型直接返回

### GroupingExpr

直接返回内部表达式的求值结果。

### UnaryExpr

- `-` 运算符：对 `double` 取负
- `!` 运算符：调用 `isTruthy()` 后取反
- `++` / `--`（前缀）：先自增/自减，再返回值
- `++` / `--`（后缀）：先返回原值，再自增/自减

### BinaryExpr

左右操作数求值后，根据运算符类型分发：

| 运算符 | 操作 |
|--------|------|
| `+` | 数字加法 / 字符串拼接 |
| `-` | 数字减法 |
| `*` | 数字乘法 |
| `/` | 数字除法（除零抛异常） |
| `%` | 数字取模（`fmod`） |
| `**` | 幂运算（`pow`） |
| `<`, `<=`, `>`, `>=` | 比较 |
| `==`, `!=` | 相等性 |

**类型提升规则**：
- `int64 + float64` / `float64 + int64` → 结果提升为 `float64`
- `string + string` → 字符串拼接
- `array + value` → 数组追加（`value` 推入数组末尾）

### 复合赋值（Compound Assignment）

`+=`, `-=`, `*=`, `/=`, `%=` 在 `AssignmentExpr` 中处理，等价于对应的二元运算 + 赋值：

```vora
x += 1    // 等价于 x = x + 1
s += "!"  // 等价于 s = s + "!"（字符串拼接）
arr += [3, 4]  // 等价于 arr = arr + [3, 4]（数组追加）
```

### PropertyExpr

```
evaluate(object)
→ 检查是 ObjectInstance
→ 查找 properties 表
→ 未找到则查找 classDefinition->methods 表
→ 返回 BoundMethodCallable（绑定 this）
→ 都没有则抛出 "Undefined property"
```

**BoundMethodCallable**：在 `evaluate()` 内部定义的匿名类，继承 `Callable`。调用时：
1. push 闭包作用域
2. 定义参数
3. 定义 `this` 为当前实例
4. 执行方法体
5. 捕获 `ReturnSignal` 返回

### PropertyAssignmentExpr

```
evaluate(object) + evaluate(value)
→ 检查是 ObjectInstance
→ instance->properties[prop] = value
→ 返回 value
```

### ThisExpr

```
环境.get("this", keyword)
→ 找不到则报 "Cannot use 'this' outside of object method"
```

## execute() — 语句执行

### BlockStmt

```
pushScope() → 执行所有语句 → popScope()
```

如果遇到 `ReturnSignal`，先 popScope 再重新抛出。

### IfStmt

```
if isTruthy(evaluate(condition))
  → execute(thenBranch)
else if elseBranch
  → execute(elseBranch)
```

### WhileStmt

```
while isTruthy(evaluate(condition))
  → try { execute(body) }
    catch BreakSignal → break
    catch ContinueSignal → continue
```

### ForStmt

```
evaluate(iterable)
→ 如果是 string:
    for each character (as string):
      pushScope() → define(variable, char) → execute(body) → popScope()
→ 如果是 Array:
    for each element:
      pushScope() → define(variable, element) → execute(body) → popScope()
→ 其他类型: 抛出 RuntimeError

body 执行中捕获 BreakSignal → break，ContinueSignal → continue
```

### ReturnStmt

```
throw ReturnSignal{evaluate(value)}
```

### FuncStmt

```
创建 VoraFunction(name, params, body, captureClosure())
defineBinding(name, function)
```

`captureClosure()`：如果 scopeStack 为空则返回 `globals`，否则调用 `Environment::snapshot()` 深拷贝当前环境链。

### ObjStmt

```
1. 创建 ObjectClass：
   - className, params, body
   - 遍历 methods，为每个 FuncStmt 创建 VoraFunction（带闭包）

2. 创建 VoraFunction 作为构造函数

3. 创建 ObjectConstructorCallable（内联匿名类）：
   - 持有 classDef 和 constructorFn
   - call() 中创建 ObjectInstance，执行构造函数体（注入 this）

4. defineBinding(name, constructorCallable)
```

**ObjectConstructorCallable::call()** 逻辑：
```
创建 ObjectInstance（设置 className 和 classDefinition）
executeBlockWithThis(constructorBody, instance, params, arguments)
  → pushScope()
  → 定义参数
  → 定义 "this" = instance
  → 执行构造函数体（this.name = name 等）
  → popScope()
返回 instance
```

### LetStmt

```
defineBinding(name, evaluate(initializer))
// 若包含类型标注 (:int / :float)，typeAnnotation 字段记录标注类型
// 当前标注仅作文档用途，运行时不强制校验
```

`defineBinding()`：如果 scopeStack 为空则定义到 `globals`，否则定义到当前 `environment`。

### ExprStmt

```
result = evaluate(expression)
if REPL 模式:
    printValue(result)  // REPL 自动打印表达式结果
```

## 字符串插值

`interpolateString()` 解析 `${...}` 语法：

```
"Hello ${name}, you are ${this.age} years old"
       ↑                ↑
       查找变量 name     支持点号属性访问
```

**处理流程**：
1. 扫描字符串，遇到 `${` 开始插值
2. 提取变量名（直到 `}`）
3. 如果变量名不含 `.` → `environment.get(name)`
4. 如果变量名含 `.` → 逐级属性查找（支持属性和方法）
5. 将值转为字符串：
   - `double`：整数去掉小数部分，小数保留
   - `string`：直接拼接
   - `bool`：`"true"` / `"false"`
   - `null`：`"null"`
   - 其他：`"[object]"`
6. 变量未找到时，保留原始 `${...}` 文本（不报错）

## isTruthy() — 真值判断

```cpp
static bool isTruthy(const Value& value) {
    if (nullptr_t) return false;
    if (bool) return value;
    return true;  // 所有其他值为真
}
```

## 扩展指南

### 添加新的内建函数

在 `Interpreter` 构造函数中调用：
```cpp
defineNative("functionName", 参数个数,
    [](const std::vector<Value>& args) -> Value {
        // 实现逻辑
        return result;
    }
);
```

### 添加新的运算符求值

在 `evaluate()` 的 `BinaryExpr` 分支中，在 `switch` 语句添加新 case。

### 添加新的语言特性

1. 在 AST 中定义新节点
2. 在 Parser 中添加解析逻辑
3. 在 `evaluate()` 或 `execute()` 中添加对应的执行逻辑
