## 1. 类型系统：Int 和 Float 是两种类型

Vora 的数值类型**不是** JavaScript 的单一 `Number`，也不是 Python 的无限精度 `int`。它有**两种独立的数值类型**：

| 特性 | JavaScript | Python | Vora |
|------|-----------|--------|------|
| 整数类型 | `Number` (64-bit float) | `int` (任意精度) | `int` (64-bit signed) |
| 浮点类型 | 同上 | `float` (64-bit) | `float` (64-bit) |
| `type(1)` | `"number"` | `"int"` | `"int"` |
| `type(1.0)` | `"number"` | `"float"` | `"float"` |
| 整数溢出 | 精度丢失 | 无限制 | 回绕 (64-bit) |

### 算术运算的类型规则

```vora
// 同类型运算 → 保持类型
3 + 2        // 5      (int)
3.0 + 2.0    // 5.0    (float)

// 混合运算 → 提升为 float
3 + 2.0      // 5.0    (float)
3.0 * 4      // 12.0   (float)

// 除法 / 永远返回 float（与 Python 3 一致）
10 / 4       // 2.5    (float)
3 / 2        // 1.5    (float) — 不是 1 ！

// 整数除法用 int() 截断
int(10 / 4)  // 2      (int)

// 取模 % 保持符号
-17 % 5      // 3      (与 Python 一致)
```

```vora
// ⚠️ 注意：两个 int 相加永远是 int
// 与 JavaScript 不同（JS 中 0.1 + 0.2 是浮点精度问题）
0.1 + 0.2    // 0.3 (float) — Vora 使用 double，仍然有 IEEE 754 精度问题
```

### 字符串与 Unicode

```vora
// Vora 使用 UTF-8 编码。len() 返回的是字节数，不是字符数。
let ascii = "hello"
len(ascii)          // 5 — 每个 ASCII 字符占 1 字节

let chinese = "你好"
len(chinese)        // 6 — 每个中文字符占 3 字节（UTF-8）

// substring 也是按字节截取 ⚠️
"你好吗".substring(0, 3)  // "你" — 第一个汉字的 3 字节
```

## 2. Falsy 值：空数组和空字典也是 falsy

这是 Vora 与 **JavaScript 最大的差异之一**。Vora 的 falsy 值列表与 **Python 一致**：

```vora
// JavaScript: [] 和 {} 是 truthy  →  if ([]) { ... } 会执行
// Vora:      [] 和 {} 是 falsy   →  if ([]) { ... } 不会执行

if ([])     { print("JS 风格") }  // ❌ Vora 不会输出
if ({})     { print("JS 风格") }  // ❌ Vora 不会输出
if (null)   { print("...") }      // ❌ 不输出
if (false)  { print("...") }      // ❌ 不输出
if (0)      { print("...") }      // ❌ 不输出
if (0.0)    { print("...") }      // ❌ 不输出
if ("")     { print("...") }      // ❌ 不输出

if ([1])    { print("ok") }       // ✅ 输出 — 非空数组是 truthy
if ({a:1})  { print("ok") }       // ✅ 输出 — 非空字典是 truthy
if (-1)     { print("ok") }       // ✅ 输出 — 非零是 truthy
```

**对比速查表：**

| 值 | JS | Python | Vora |
|----|-----|--------|------|
| `null`/`nil`/`None` | falsy | falsy | **falsy** |
| `false` | falsy | falsy | **falsy** |
| `0` / `0.0` | falsy | falsy | **falsy** |
| `""` | falsy | falsy | **falsy** |
| `[]` | **truthy** | falsy | **falsy** |
| `{}` | **truthy** | falsy | **falsy** |

## 3. 变量：let/const 与类型注解

### 基本声明

```vora
let x = 42          // 可变绑定
const PI = 3.14159  // 不可变绑定 (v0.20+)
```

**注意：`const` 是浅层的。** 只锁定绑定，不锁定值：

```vora
const arr = [1, 2, 3]
arr[1] = 99         // ✅ 允许 — 修改数组内容
arr = [4, 5, 6]     // ❌ 编译错误 — 不能重新绑定

const obj = {name: "Vora"}
obj.name = "V2"     // ✅ 允许 — 修改对象属性
obj = {}            // ❌ 编译错误
```

对比：
- **JavaScript**: `const` 同样是浅层，行为一致
- **Python**: 没有真正的常量，约定 `UPPER_CASE` 是全大写

### 块级作用域

```vora
// Vora（与 JS 一致）：let/const 是块级作用域
{
    let x = 1
}
print(x)  // ❌ RuntimeError: Undefined variable 'x'

// Vora 没有全局作用域提升（hoisting）
print(y)  // ❌ RuntimeError: Undefined variable 'y'
let y = 2
```

对比：
- **JavaScript**: `let`/`const` 有暂时性死区（TDZ），访问前报 `ReferenceError`
- **Python**: 函数作用域 + 模块作用域，没有块级作用域
- **Lua**: `local` 是块级，`global` 默认

### 类型注解：运行时类型转换

Vora 支持 `:int`、`:float`、`:bool`、`:str` / `:string` 注解，但**是运行时值转换，不是编译时类型检查**！

```vora
let a:float = 1         // → 1.0 (float)  — int 自动转为 float
let b:int = 3.14        // → 3 (int)      — float 截断为 int
let c:bool = "hello"    // → true (bool)  — 非空字符串 → true
let d:str = 42          // → "42" (str)   — int 转为字符串

// ⚠️ 已知问题：字符串到数值的转换目前有 bug
let e:int = "42"        // → 0 (bug! 应该得到 42)
// 解决方法：使用显式转换
let e:int = int("42")   // ✅ 正确：42
```

对比：
- **TypeScript**: 编译时类型检查，零运行时开销
- **Python**: 类型注解仅用于静态分析工具（mypy），完全不改变运行时行为
- **Vora**: 运行时主动转换 — 这是有意为之的设计，不是 bug

### 解构赋值

```vora
// 数组解构
let [a, b] = [1, 2]         // a=1, b=2
let [x, ...rest] = [1,2,3]  // x=1, rest=[2,3]

// 对象解构
let {name, age} = person    // name=person.name, age=person.age

// 嵌套解构
let {user: {profile: {bio}}} = data

// 解构中的默认值
let [val = 99] = []         // val=99
```

## 4. 运算符：没有位运算，&&/|| 返回值

### 逻辑运算符返回值

Vora 的 `&&` 和 `||` 返回**实际值**，不是布尔值。与 JavaScript 和 Python 一致：

```vora
// || 返回第一个 truthy 值，或最后的 falsy 值
null || "fallback"   // "fallback"
0 || 42              // 42
"" || "default"      // "default"
1 || 2               // 1 (first truthy)

// && 返回第一个 falsy 值，或最后的 truthy 值
"hi" && 42           // 42
null && "anything"   // null
1 && 2 && 3          // 3 (last truthy)
0 && "anything"      // 0 (first falsy)

// 典型用法：安全访问默认值
let name = userInput || "Guest"
let enabled = flag && checkPermission()
```

### ?? 空值合并（v0.21+）

`??` 只检查 `null`，不检查其他 falsy 值。这与 JavaScript 的 `??` 行为一致，但不同于 `||`：

```vora
// ?? 只关心 null
0 ?? 100    // 0    — 0 不是 null
"" ?? "x"   // ""   — 空字符串不是 null
null ?? 42  // 42   — null 触发替代

// 对比：|| 关心所有 falsy
0 || 100    // 100  — 0 是 falsy
"" || "x"   // "x"  — "" 是 falsy
```

```javascript
// JavaScript 对比：行为完全一致
0 ?? 100    // 0
0 || 100    // 100
```

### ?. 可选链（v0.21+, v0.25 增强）

```vora
let street = user?.address?.street   // null if any step is null
let result = obj?.method?.(arg)      // 安全调用方法
let item = arr?.[index]              // 安全索引访问（越界也返回 null）

// v0.25+: dict 不存在的 key 也返回 null
let d = {a: 1}
d?.nonexistent  // null（之前会抛 RuntimeError）
```

### 没有位运算符

Vora **不提供** `&`、`|`、`^`、`~`、`<<`、`>>`。这是有意为之的设计选择。

```vora
// JavaScript / Python:
let flags = READ | WRITE          // 位或
if (flags & READ) { ... }         // 位与
let doubled = value << 1          // 左移

// Vora: 用算术或函数替代
// 位运算场景通常通过 std 模块或 native 函数处理
```

### 三元运算符右结合

```vora
// 三元是右结合的（与 JS 一致）
a ? b : c ? d : e    →    a ? b : (c ? d : e)
```

## 5. 函数：命名参数、剩余参数、展开调用

### 基本语法

```vora
func add(a, b) {
    return a + b
}

// 匿名函数 / lambda（没有箭头函数语法）
let double = func(x) { return x * 2 }
```

对比：

| 特性 | Vora | JavaScript | Python |
|------|------|------------|--------|
| 声明函数 | `func f() {}` | `function f() {}` | `def f():` |
| 匿名函数 | `func(x) { ... }` | `(x) => { ... }` | `lambda x: ...` |
| 块语法 | 大括号 `{}` | 大括号 `{}` | 缩进 + `:` |

### 默认参数（可引用先前参数）

```vora
func addOne(a, b = a + 1) {
    return b
}
addOne(5)       // 6 — b 默认 = a + 1
addOne(5, 10)   // 10 — 显式传入覆盖默认值

// 默认值可以是任意表达式
func compute(a, b = a * 2, c = a + b) {
    return [a, b, c]
}
compute(3)      // [3, 6, 9] — b = 3*2, c = 3+6
compute(3, 10)  // [3, 10, 13] — b=10, c=3+10
```

对比：
- **JavaScript**: 同样支持默认值引用先前参数
- **Python**: 默认值在**定义时**求值（可变默认值的经典陷阱），Vora 在**调用时**求值

### 剩余参数（Rest Parameters）

```vora
// ...name 必须是最后一个参数
func sumAll(...nums) {
    let total = 0
    for n in nums { total = total + n }
    return total
}
sumAll(1, 2, 3, 4)   // 10
sumAll()              // 0 — 无参数时为空数组 []

// 混合固定参数和剩余参数
func greet(greeting, ...names) {
    // greeting = "Hi", names = ["Alice", "Bob"]
    for n in names { print(greeting + ", " + n) }
}
greet("Hi", "Alice", "Bob")
```

### 命名参数（v0.26+）

```vora
func divide(dividend, divisor) {
    return dividend / divisor
}

// 按名称传参，顺序随意
divide(dividend=100, divisor=4)  // 25
divide(divisor=2, dividend=50)   // 25

// 位置参数 + 命名参数混用（位置必须在前面）
func draw(x, y, color, size) { ... }
draw(10, 20, color="red", size=3)      // ✅
draw(10, 20, size=5, color="blue")     // ✅
draw(color="red", 10, 20)              // ❌ 命名参数不能在位置参数之前
```

对比：
- **Python**: `func(a, b, *, c, d)` — 需要用 `*` 标记命名关键字参数边界
- **Vora**: 不需要 `*`，所有参数都可以按名称传递
- **JavaScript**: 没有原生命名参数，通常用解构对象模拟 `func({a, b})`

### 调用展开（Call Spread）

```vora
// ...arr 展开数组为独立参数
func threeArgs(a, b, c) {
    return a + b + c
}
threeArgs(...[1, 2, 3])  // 6

// 多个展开 + 固定参数混用
sumAll(...part1, ...part2, 42, ...part3)

// 典型的转发模式：rest 收 → spread 传
func forward(prefix, ...rest) {
    return someFunc(prefix, ...rest)
}
```

### 递归与尾调用优化

Vora v0.20+ 支持尾调用优化（TCO）。函数的最后一个操作是直接调用另一个函数时，Vora 会复用当前栈帧：

```vora
// ✅ 会被优化：return 后直接调用自身
func factorial(n, acc = 1) {
    if (n <= 1) return acc
    return factorial(n - 1, acc * n)  // TCO — 不增加栈深度
}

// ✅ 会被优化：两个分支都是尾调用
func even(n) {
    if (n == 0) return true
    return odd(n - 1)   // TCO
}
func odd(n) {
    if (n == 0) return false
    return even(n - 1)  // TCO
}

// ❌ 不会被优化：调用后还有乘法操作
func fact(n) {
    if (n <= 1) return 1
    return n * fact(n - 1)  // 需要保留 n 做乘法，不能 TCO
}
```

## 6. 闭包：捕获并可变

Vora 的闭包**可以修改**外层变量。这比 Python 更直观：

```vora
func makeCounter() {
    let count = 0
    func next() {
        count = count + 1    // ✅ 修改外层的 count
        return count
    }
    return next
}

let c = makeCounter()
c()  // 1
c()  // 2
c()  // 3
```

```python
# Python 对比：需要用 nonlocal 声明
def make_counter():
    count = 0
    def next():
        nonlocal count   # 必须声明，否则 count += 1 会报错
        count += 1
        return count
    return next
```

```javascript
// JavaScript 对比：不需要声明，与 Vora 行为一致
function makeCounter() {
    let count = 0
    return function() {
        count += 1
        return count
    }
}
```

### 多个闭包共享状态

```vora
func makePair() {
    let n = 0
    func inc() { n = n + 1; return n }
    func dec() { n = n - 1; return n }
    return [inc, dec]     // 返回数组
}
let [up, down] = makePair()
up()    // 1
up()    // 2
down()  // 1  — 共享同一个 n
```

## 7. 对象模型：Obj 关键字 + C3 多继承

Vora 的对象模型是它与 JavaScript、Python、Lua **最大的差异化特性**。

### 基本语法

```vora
Obj Person(name, age) {
    this.name = name
    this.age = age
    func greet() {
        print("I'm " + this.name + ", age " + this.age)
    }
    func birthday() {
        this.age = this.age + 1
    }
}

let alice = Person("Alice", 25)   // 不需要 new 关键字
alice.greet()                      // "I'm Alice, age 25"
alice.birthday()                   // age → 26
```

**关键设计：**

| 特性 | Vora | JavaScript (ES6) | Python |
|------|------|-------------------|--------|
| 定义类 | `Obj Foo(a) { ... }` | `class Foo { constructor(a) {...} }` | `class Foo: def __init__(self, a):` |
| 构造函数 | 类名后面的参数列表 | `constructor()` 方法 | `__init__()` 方法 |
| 实例化 | `Foo(a)` **无 new** | `new Foo(a)` | `Foo(a)` |
| `this` 绑定 | 自动绑定，无歧义 | 函数可重绑定，常见 bug 来源 | `self` 显式作为首参 |
| 属性声明 | 构造器中 `this.x = val` | 同上 | 同上（通常在 `__init__`） |

### 单继承

```vora
Obj Animal(name) {
    this.name = name
    func speak() { return this.name + " makes a sound" }
}

Obj Dog : Animal(name, breed) {
    this.breed = breed
    func speak() { return this.name + " barks" }   // 覆盖父类方法
}
```

### super 调用

```vora
Obj Base(name) {
    this.name = name
    func who() { return "Base: " + this.name }
}
Obj Child : Base(name) {
    func who() { return "Child: " + super.who() }  // super 调用父类方法
}
```

### 多继承 + C3 MRO（v0.15+）

Vora 使用与 Python 完全相同的 **C3 线性化算法**：

```vora
Obj A() {
    func tag() { return "A" }
}
Obj B : A() {
    func tag() { return "B" }
}
Obj C : A() {
    func tag() { return "C" }
}
Obj D : B, C() {}    // 钻石继承

let d = D()
d.tag()              // "B" — C3 MRO 线性化
```

```python
# Python 对比：行为完全一致
class A:
    def tag(self): return "A"
class B(A):
    def tag(self): return "B"
class C(A):
    def tag(self): return "C"
class D(B, C): pass   # Python 也使用 C3 MRO

D().tag()  # "B" — 与 Vora 一致
```

**构造器调用顺序**：从根到叶，逆向 MRO 顺序（与 Python 的 `super().__init__()` 链效果一致）：

```vora
Obj Base(name) { this.name = name; this.chain = ["Base"] }
Obj Middle : Base(name, value) { this.value = value; this.chain.add("Middle") }
Obj Leaf : Middle(name, value, extra) { this.extra = extra; this.chain.add("Leaf") }

let leaf = Leaf("root", 42, "extra")
leaf.chain  // ["Base", "Middle", "Leaf"] — 构造器从根到叶子逐层执行
```

**注意**：Vora 的构造器链**自动执行**，不需要手动调用 `super(...)`。每个父类构造器的参数从子类的参数列表中**按名称/位置匹配**。

### 静态方法：this.func（v0.25+）

Vora 用 `this.func` 声明静态方法——**不加新关键字**，复用已有的 `this` + `func`（学习 Ruby 的 `def self.method`）：

```vora
Obj MathUtil() {
    this.func square(x) { return x * x }
}

MathUtil.square(5)  // 25 — 挂在类上，不需要实例

Obj Point(x, y) {
    this.x = x; this.y = y
    this.func origin() { return Point(0, 0) }   // 静态工厂
    func len() { return MathUtil.square(this.x) + MathUtil.square(this.y) }
}

Point.origin().len()  // 0 — 类调用静态方法
```

| 特性 | Vora | JavaScript | Python |
|------|------|-----------|--------|
| 实例方法 | `func f()` — 隐式 `this` | `f() { this.x }` | `def f(self):` — 显式 `self` |
| 静态方法 | `this.func f()` — 无 `this` | `static f() {}` | `@staticmethod` / `@classmethod` |
| 类调用 | `Class.f()` | `Class.f()` | `Class.f()` |
| 实例调用 | ✅ `obj.f()` | ✅ `obj.f()` | ✅ `obj.f()` |

## 8. 控制流：match 表达式、defer、do-while

### match 表达式（v0.24+）

Vora 的 `match` 是**表达式**，会返回值。支持字面量、通配符、范围模式：

```vora
// 基本匹配
let name = match n {
    0 => "zero",
    1 => "one",
    _ => "other"       // _ 是通配符
}

// 范围匹配
let grade = match score {
    90..=100 => "A",   // ..= 包含上限 (inclusive)
    80..=89  => "B",
    70..=79  => "C",
    60..=69  => "D",
    _        => "F"
}

let result = match x {
    1..5 => "low",     // .. 不含上限 (exclusive)
    _    => "high"
}

// 块体（block body）
let value = match x {
    1 => {
        let t = compute()
        t * 2           // 最后一个表达式是返回值
    },
    _ => 0
}

// 函数中使用
func describe(n) {
    return match n {
        0 => "zero",
        1 => "one",
        _ => "many"
    }
}
```

对比：
- **Rust**: `match x { 0 => "zero", _ => "other" }` — 几乎相同
- **Python 3.10+**: `match x: case 0: return "zero"` — 语法不同
- **JavaScript**: 无原生 match，通常用 `switch` 或对象映射
- **Vora 不支持**: 多模式 `1 | 2 => "a"`、守卫 `x if x > 0 => "pos"`、解构模式（这些在路线图中）

### defer：延迟执行

`defer` 在**正常返回**时按 LIFO 顺序执行（后进先出）：

```vora
func demo() {
    defer func() { print("2nd") }()
    defer func() { print("1st") }()
    print("0")
    return
}
demo()
// 输出：0 → 1st → 2nd

// 典型用法：资源清理
func readConfig(path) {
    let file = openFile(path)
    defer func() { file.close() }()   // 无论哪里 return，都会关闭文件
    // ... 处理文件 ...
    return result
}
```

对比：
- **Go**: `defer file.Close()` — LIFO，return 和 panic 都会执行
- **Vora**: `defer func() { ... }()` — 同函数内 throw 前会执行（v0.25），跨函数异常展开暂不执行
- **Python**: 用 `with` / `try-finally` 替代

### do-while（v0.25+）

```vora
// 与 C/Java/JS 语义一致：先执行一次，再检查条件
let i = 0
do {
    print(i)
    i = i + 1
} while (i < 3)
// 输出 0, 1, 2
```

## 9. 异常处理：throw 任意值

### throw 可以抛出任何类型

```vora
throw "something went wrong"         // 字符串
throw 404                            // 数字
throw {code: 500, msg: "internal"}   // 对象
throw Error("typed error")           // 自定义对象
```

```javascript
// JavaScript 对比：可以 throw 任何值，行为一致
throw "error"      // ✅
throw 404          // ✅
throw new Error()  // ✅ 但推荐 Error 对象
```

```python
# Python 对比：只能 throw Exception 的子类
raise "error"      # ❌ TypeError: exceptions must derive from BaseException
raise Exception()  # ✅ 必须
```

### catch 捕获所有异常

Vora 的 `catch` **没有类型过滤**，所有异常都进入同一个处理块：

```vora
try {
    riskyOperation()
} catch (e) {
    // e 可能是任何类型：string, int, object...
    if (type(e) == "string") {
        print("String error: " + e)
    } else {
        print("Unknown error type")
    }
}
```

```javascript
// JavaScript 对比：catch 可以按类型过滤（ES2019+）
try { ... } catch (e) {
    if (e instanceof TypeError) { ... }
    else if (e instanceof RangeError) { ... }
}
```

### try/catch/finally 完整示例

```vora
// finally 始终执行，包括 return 时
func demo() {
    try {
        print("A")
        return "value"
    } finally {
        print("B")    // return 之前执行
    }
}
demo()
// 输出：A → B
// 返回值："value"

// 嵌套 try
try {
    try {
        throw "inner"
    } catch (inner) {
        print("Caught inner:", inner)
    }
    print("After inner")   // ✅ 会执行
} finally {
    print("Outer finally") // ✅ 会执行
}

// 异常传播
func propagate() {
    try {
        let x = 1 / 0      // 除零错误
    } finally {
        print("cleanup")    // ✅ 执行清理
    }
    print("after")          // ❌ 不执行
}
try {
    propagate()
} catch (e) {
    print("Caught propagated:", e)  // ✅ 捕获
}
```

## 10. 集合：引用相等与字符串键

### 数组和字典使用引用相等

这是 Vora 与 Python **最大的差异之一**：

```vora
[1, 2] == [1, 2]          // false — 不同对象，引用不同
{a: 1} == {a: 1}          // false — 同上

// 基本类型才用值相等
42 == 42                  // true
"hello" == "hello"        // true
true == true              // true
null == null              // true

// 跨类型数值相等
42 == 42.0                // true — int 等于 float
```

```python
# Python 对比：集合使用值相等
[1, 2] == [1, 2]          # True
{"a": 1} == {"a": 1}      # True
```

```javascript
// JavaScript 对比：与 Vora 行为一致
[1, 2] === [1, 2]          // false
{a: 1} === {a: 1}          // false
```

**手动比较数组：**

```vora
func arraysEqual(a, b) {
    if (len(a) != len(b)) return false
    for (let i = 0; i < len(a); i = i + 1) {
        if (a[i] != b[i]) return false
    }
    return true
}
```

### 字典键必须是字符串

```vora
let d = {}
d[42] = "answer"          // 键自动转为 "42"
d[true] = "bool key"      // 键自动转为 "true"

```

### 数组和字典的方法

```vora
// 数组方法

arr.add(4)               // [1,2,3,4] — 追加
arr.insert(0, 0)         // [0,1,2,3,4] — 在位置 0 插入
```

## 12. 生成器：yield + iter/next 协议

```vora
func countTo(n) {
    for (let i = 1; i <= n; i = i + 1) {
        yield i
    }
}

// 手动消费生成器
let gen = iter(countTo(3))
next(gen)  // 1
next(gen)  // 2
next(gen)  // 3
// 耗尽后再调用 next() = StopIteration 异常

// 用 try/catch 循环消费
let it = iter(countTo(5))
while (true) {
    try {
        let val = next(it)
        print(val)
    } catch (e) {
        break   // StopIteration
    }
}

// v0.25+: for-in 可直接消费生成器
for v in countTo(5) { print(v) }  // 1, 2, 3, 4, 5
```

对比：
- **Python**: `yield` + `for x in gen()` — for 循环直接支持
- **JavaScript**: `function*` + `yield` + `for...of` — 类似
- **Vora**: `yield` + `for v in gen()` — 与 Python 行为一致（v0.25）
