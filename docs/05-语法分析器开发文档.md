# 语法分析器（Parser）开发文档

## 概述

语法分析器将 Token 流转换为抽象语法树（AST）。Vora 使用 **Pratt 解析器**（优先级爬升法）处理表达式，使用 **递归下降** 处理语句。

- **相关文件**：`src/parser/parser.h`、`src/parser/parser.cpp`
- **命名空间**：`vora`
- **行数**：`.h` 71 行，`.cpp` ~1114 行

## Parser 类

### 构造

```cpp
explicit Parser(std::vector<Token> tokens);
```

接收词法分析器输出的 Token 流。

### 公开接口

```cpp
std::unique_ptr<Program> parse();
```

解析入口。循环调用 `statement()` 直到文件末尾，组装 `Program` 节点返回。任何语句解析失败返回 `nullptr` 时，整个 `parse()` 也返回 `nullptr`。

## 表达式解析

### 优先级表

Pratt 解析器的核心——运算符优先级：

| 优先级 | 运算符 | 说明 | 结合性 |
|--------|--------|------|--------|
| 1 | `=`, `\|\|` | 赋值、逻辑或 | `=` 右结合 |
| 2 | `&&`, `? :` | 逻辑与、三元条件 | `? :` 右结合 |
| 3 | `==`, `!=` | 相等性比较 | 左结合 |
| 4 | `<`, `<=`, `>`, `>=` | 关系比较 | 左结合 |
| 5 | `+`, `-` | 加减 | 左结合 |
| 6 | `*`, `/`, `%` | 乘除模 | 左结合 |
| 7 | `**` | 幂运算 | 右结合 |
| 0 | 其他 | 不是中缀运算符 | — |

### 解析流程

```
expression()
  └── parsePrecedence(0)         ← Pratt 核心
        ├── call()               ← 解析左操作数
        │     ├── primary()      ← 原子表达式
        │     └── 循环：() [] .  ← 后缀操作
        └── while (当前优先级 > 传入优先级)
              ├── 消费运算符
              ├── parsePrecedence(右操作数优先级)
              └── 构造 BinaryExpr / AssignmentExpr
```

### primary() — 原子表达式解析

| 当前 Token | 生成的节点 | 示例 |
|------------|-----------|------|
| `MINUS` | `UnaryExpr` | `-42` |
| `NOT` | `UnaryExpr` | `!flag` |
| `THIS` | `ThisExpr` | `this` |
| `NUMBER` | `LiteralExpr(int64_t 或 double)` | `42`→int64, `3.14`→float64 |
| `STRING` | `LiteralExpr(string)` | `"hello"` |
| `TRUE` | `LiteralExpr(true)` | `TRUE` |
| `FALSE` | `LiteralExpr(false)` | `FALSE` |
| `NULL_TOKEN` | `LiteralExpr(nullptr)` | `null` |
| `IDENTIFIER` | `VariableExpr` | `x` |
| `LEFT_BRACKET` | `ArrayExpr` | `[1, 2]` |
| `LEFT_PAREN` | `GroupingExpr` | `(1 + 2)` |

### call() — 后缀操作解析

`call()` 在 `primary()` 之后循环处理后缀操作，支持链式调用：

| 下一个 Token | 操作 | 生成的节点 | 示例 |
|-------------|------|-----------|------|
| `(` | 函数调用 | `CallExpr` | `f(a, b)` |
| `[` | 索引访问 | `IndexExpr` | `arr[0]` |
| `.` | 属性访问 | `PropertyExpr` | `obj.name` |

这些操作可以链式组合：`obj.method(arr[0]).prop`

### parsePrecedence() — Pratt 核心

```cpp
std::unique_ptr<Expr> Parser::parsePrecedence(int precedence) {
    auto left = call();  // 解析左操作数

    while (!isAtEnd() && precedence < getPrecedence(peek().type)) {
        Token op = advance();
        int opPrec = getPrecedence(op.type);

        // 三元运算符特殊处理（优先级 2）
        if (op.type == QUESTION) {
            auto thenBranch = parsePrecedence(0);
            consume(COLON, "Expected ':' in ternary expression");
            // 右结合：else 分支以 opPrec-1 解析
            auto elseBranch = parsePrecedence(opPrec - 1);
            left = make_unique<TernaryExpr>(left, thenBranch, elseBranch);
            continue;
        }

        // 处理右结合（** 和 =）
        int nextPrec = opPrec - (rightAssociative ? 1 : 0);
        auto right = parsePrecedence(nextPrec);

        if (op.type == EQUAL) {
            // 赋值特殊处理：检查左侧是否为合法目标
            // VariableExpr → AssignmentExpr
            // PropertyExpr → PropertyAssignmentExpr
        } else {
            left = make_unique<BinaryExpr>(left, op, right);
        }
    }
    return left;
}
```

**赋值的特殊处理**：在 Pratt 循环中，当遇到 `=` 运算符时，检查左侧是否为 `VariableExpr` 或 `PropertyExpr`，分别构造 `AssignmentExpr` 或 `PropertyAssignmentExpr`。如果左侧不合法则报错。

**三元运算符的特殊处理**：`? :` 在 Pratt 循环中作为中缀运算符处理（优先级 2）。当遇到 `?` 时，解析 then 分支（优先级 0，即整个表达式），消费 `:`，再解析 else 分支（优先级 `opPrec-1`，实现右结合）。短路求值在解释器端实现。

## 语句解析

### statement() — 语句分发

```
statement()
  ├── LET      → letStatement()
  ├── {        → blockStatement()
  ├── RETURN   → returnStatement()
  ├── IF       → ifStatement()
  ├── WHILE    → whileStatement()
  ├── FOR      → forStatement()
  ├── FUNC     → funcStatement()
  ├── OBJ      → objStatement()
  ├── BREAK    → breakStatement()
  ├── CONTINUE → continueStatement()
  ├── TRY      → tryStatement()
  ├── THROW    → throwStatement()
  └── 默认     → expression() → ExprStmt
```

### 各语句解析方法

#### letStatement()

语法：`let <name> = <expression> ;`

```
let x = 10
    │   │
    │   └── expression()
    └── IDENTIFIER
```

生成 `LetStmt(name, initializer)`。

#### blockStatement()

语法：`{ <statement>* }`

在 `{` 已被消费的前提下，循环解析语句直到 `}`。生成 `BlockStmt(statements)`。

#### returnStatement()

语法：`return <expression> ;`

解析 `return` 后的表达式。生成 `ReturnStmt(value)`。

#### ifStatement()

语法：`if ( <expression> ) <statement> ( else <statement> )?`

1. 消费 `(`，解析条件表达式，消费 `)`
2. 解析 then 分支
3. 如果有 `else`，解析 else 分支

生成 `IfStmt(condition, thenBranch, elseBranch)`。

#### whileStatement()

语法：`while ( <expression> ) <statement>`

生成 `WhileStmt(condition, body)`。

#### forStatement()

语法：`for <identifier> in <expression> { <block> }`

1. 消费循环变量名
2. 消费 `in`
3. 解析可迭代表达式
4. 消费 `{`，调用 `blockStatement()`

生成 `ForStmt(variable, iterable, body, forToken)`。

#### funcStatement()

语法：`func <name> ( <params> ) { <block> }`

1. 消费函数名
2. 消费 `(`，解析逗号分隔的参数列表，消费 `)`
3. 消费 `{`，调用 `blockStatement()`
4. 使用 `dynamic_cast` 提取 `BlockStmt`，转为 `shared_ptr`

生成 `FuncStmt(name, params, body)`。

#### objStatement()

语法：`Obj <name> (: <Parent>)? ( <params> ) { <body+methods> }`

1. 消费类名
2. **可选**：消费 `:` 和父类名（继承）
3. 解析构造函数参数列表
4. 解析块内语句：
   - `FuncStmt` → 归入 `methods`
   - 其他语句 → 归入 `body`（构造函数体）
5. 将 body 语句打包为 `shared_ptr<BlockStmt>`

生成 `ObjStmt(name, params, methods, body)`。

#### breakStatement() / continueStatement()

语法：`break` / `continue`

简单的关键字消费，生成 `BreakStmt(keyword)` / `ContinueStmt(keyword)`。语义校验（是否在循环内）推迟到解释器运行时。

#### throwStatement()

语法：`throw <expression>`

解析 `throw` 后的表达式。生成 `ThrowStmt(value, keyword)`。

#### tryStatement()

语法：`try { <block> } catch ( <identifier> ) { <block> } ( finally { <block> } )?`

或：`try { <block> } finally { <block> }`（无 catch）

1. 消费 `{`，调用 `blockStatement()` 解析 try 块
2. 如果有 `catch`：消费 `(`、捕获变量名、`)`、`{`，解析 catch 块
3. 如果有 `finally`：消费 `{`，解析 finally 块
4. catch 和 finally 至少有一个

生成 `TryStmt(tryBlock, catchVar, catchBlock, finallyBlock)`。`catchVar` 为空字符串表示无 catch，`catchBlock`/`finallyBlock` 为 `nullptr` 表示不存在。

#### tryStatement()

语法：`try { <block> } catch ( <identifier> ) { <block> } ( finally { <block> } )?`

或：`try { <block> } finally { <block> }`（无 catch）

1. 消费 `{`，调用 `blockStatement()` 解析 try 块
2. 如果有 `catch`：消费 `(`、捕获变量名、`)`、`{`，解析 catch 块
3. 如果有 `finally`：消费 `{`，解析 finally 块
4. catch 和 finally 至少有一个

生成 `TryStmt(tryBlock, catchVar, catchBlock, finallyBlock)`。`catchVar` 为空字符串表示无 catch，`catchBlock`/`finallyBlock` 为 `nullptr` 表示不存在。

#### throwStatement()

语法：`throw <expression>`

解析 `throw` 后的表达式。生成 `ThrowStmt(value, keyword)`。

## Token 流导航工具

| 方法 | 说明 |
|------|------|
| `isAtEnd()` | 当前 Token 是否为 `END_OF_FILE` |
| `advance()` | 消费当前 Token 并返回它 |
| `peek()` | 返回当前 Token（不消费） |
| `previous()` | 返回上一个已消费的 Token |
| `match(type)` | 如果当前 Token 匹配则消费并返回 `true` |
| `check(type)` | 检查当前 Token 类型（不消费） |

## 错误处理

Parser 使用 `std::cerr` 输出错误信息并收集错误。具备 **panic mode 错误恢复**：`synchronize()` 方法在遇到错误时跳过 Token 直到同步点（`;`、`}`、语句关键字），继续解析后续代码，一次可收集多个错误。`parse()` 在所有语句解析完成后统一检查 `hadError` 判定是否失败。

常见错误消息：
- `"Expected variable name"`
- `"Expected '=' after variable name"`
- `"Expected ')' after condition"`
- `"Expected '}' after block"`
- `"Unexpected token: ..."`
- `"Invalid assignment target"`
