# 词法分析器（Lexer）开发文档

## 概述

词法分析器是 Vora 编译流水线的第一阶段，负责将源代码文本转换为 Token 序列。

- **相关文件**：`src/lexer/lexer.h`、`src/lexer/lexer.cpp`、`src/lexer/token.h`、`src/lexer/token.cpp`
- **命名空间**：`vora`

## TokenType 枚举

定义在 `token.h`，共 47 种 Token 类型：

```cpp
enum class TokenType {
    // 括号
    LEFT_PAREN, RIGHT_PAREN,        // ( )
    LEFT_BRACE, RIGHT_BRACE,        // { }
    LEFT_BRACKET, RIGHT_BRACKET,    // [ ]

    // 算术运算符
    PLUS, MINUS, MULTIPLY, DIVIDE, MODULO,  // + - * / %

    // 赋值/比较运算符
    EQUAL, LESS, GREATER,           // = < >
    PLUS_PLUS, MINUS_MINUS,         // ++ --
    POWER,                          // **
    PLUS_EQUAL, MINUS_EQUAL,        // += -=
    MULTIPLY_EQUAL, DIVIDE_EQUAL,   // *= /= 
    MODULO_EQUAL,                   // %=
    EQUAL_EQUAL, LESS_EQUAL,        // == <=
    GREATER_EQUAL, NOT_EQUAL,       // >= !=
    AND, OR, NOT,                   // && || !
    TRUE, FALSE, NULL_TOKEN,        // true false null

    // 标点
    COMMA, DOT, COLON, QUESTION, SEMICOLON,   // , . : ? ;

    // 字面量
    NUMBER, STRING, IDENTIFIER,

    // 关键字
    LET, FUNC, RETURN, IF, ELSE,   // let func return if else
    WHILE, FOR, IN, OBJ, THIS,     // while for in Obj this
    SUPER, BREAK, CONTINUE,        // super break continue
    TRY, CATCH, FINALLY, THROW,    // try catch finally throw

    // 特殊
    END_OF_FILE, INVALID
};
```

## Token 结构体

```cpp
struct Token {
    TokenType type;       // Token 类型
    std::string lexeme;   // 源码中的原始文本
    int line;             // 所在行号
    int column;           // 所在列号
};
```

**辅助函数**：
- `tokenTypeToString(TokenType)`：将枚举转为大写字符串（如 `PLUS` → `"PLUS"`）
- `Token::toString()`：格式化为 `"TYPE(lexeme)"`

## Lexer 类

### 构造

```cpp
explicit Lexer(std::string source);
```

接收完整的源代码字符串。

### 公开接口

```cpp
std::vector<Token> scanTokens();
```

扫描全部 Token，在末尾自动追加 `END_OF_FILE` Token，返回完整列表。

### 工作原理

采用 **单遍扫描（Single-Pass Scanning）** 算法：

```
source: "let x = 10"
         │
         ▼
      scanTokens() 循环
         │
         ├── start = current
         ├── scanToken()  →  消费字符，产生 Token
         └── 循环直到 isAtEnd()
```

核心字段：

| 字段 | 作用 |
|------|------|
| `start` | 当前 Token 的起始位置（用于 substr 取 lexeme） |
| `current` | 当前扫描位置 |
| `line` | 当前行号（遇 `\n` 递增） |

### scanToken() 分发表

`scanToken()` 是核心方法，根据当前字符分发到不同的处理逻辑：

| 字符 | 处理 | 产生的 Token |
|------|------|-------------|
| `(`, `)`, `{`, `}`, `[`, `]` | 直接 addToken | 对应括号类型 |
| `,`, `.`, `:` | 直接 addToken | `COMMA`, `DOT`, `COLON` |
| `;` | 直接 addToken | `SEMICOLON` |
| `+` | 看下一个是否为 `+` | `PLUS` 或 `PLUS_PLUS` |
| `-` | 看下一个是否为 `-` | `MINUS` 或 `MINUS_MINUS` |
| `*` | 看下一个是否为 `*` | `MULTIPLY` 或 `POWER` |
| `/` | 看下一个是否为 `/` 或 `*` | `DIVIDE`、行注释、块注释 |
| `%` | 直接 addToken | `MODULO` |
| `=` | 看下一个是否为 `=` | `EQUAL` 或 `EQUAL_EQUAL` |
| `<` | 看下一个是否为 `=` | `LESS` 或 `LESS_EQUAL` |
| `>` | 看下一个是否为 `=` | `GREATER` 或 `GREATER_EQUAL` |
| `!` | 看下一个是否为 `=` | `NOT` 或 `NOT_EQUAL` |
| `&` | 看下一个是否为 `&` | `AND` 或 `INVALID` |
| `|` | 看下一个是否为 `|` | `OR` 或 `INVALID` |
| `"` | 调用 `string()` | `STRING` |
| 空格, `\r`, `\t` | 跳过 | 无 |
| `\n` | `line++` | 无 |
| 数字 | 调用 `number()` | `NUMBER` |
| 字母/`_` | 调用 `identifier()` | `IDENTIFIER` 或关键字 |
| 其他 | addToken | `INVALID` |

### 数字扫描

`number()` 方法：

1. 检查 `0x`/`0o`/`0b` 前缀（必须在 `0` 之后立即出现）
   - `0x`/`0X` → 消费十六进制数字（`0-9`、`a-f`、`A-F`）
   - `0o`/`0O` → 消费八进制数字（`0-7`）
   - `0b`/`0B` → 消费二进制数字（`0-1`）
2. 否则连续消费十进制数字
3. 如果遇到 `.` 且下一个字符是数字，则消费小数部分
4. 支持整数（`42`）、小数（`3.14`）和带前缀整数（`0xFF`）

```
"123"   → NUMBER("123")
"3.14"  → NUMBER("3.14")
"3."    → NUMBER("3") + DOT（不消费孤立的小数点）
"0x1A"  → NUMBER("0x1A")  // 26
"0o12"  → NUMBER("0o12")  // 10
"0b101" → NUMBER("0b101") // 5
```

### 字符串扫描

`string()` 方法：

1. 从 `"` 之后开始，消费字符直到遇到闭合 `"` 或文件末尾
2. 支持多行字符串（遇到 `\n` 会递增行号）
3. 未闭合的字符串产生 `INVALID` Token
4. Token 的 `lexeme` 不包含引号

```
"hello"  → STRING("hello")
"unclosed → INVALID
```

### 标识符与关键字扫描

`identifier()` 方法：

1. 连续消费字母（含 Unicode）、数字、下划线
2. 在 `keywords` 哈希表中查找
3. 匹配到关键字则使用对应的 `TokenType`，否则为 `IDENTIFIER`

**`isAlpha(char)` 实现**：

```cpp
bool Lexer::isAlpha(char c) const {
    return std::isalpha(static_cast<unsigned char>(c)) || c == '_' ||
        static_cast<unsigned char>(c) > 127;  // Unicode (UTF-8 multi-byte)
}
```

任何 UTF-8 多字节序列的字节值均 > 127，因此 Unicode 字符可自然用作标识符首字符及后续字符。标识符示例：`名字`、`ユーザー名`、`café`、`用户_count`。

### 注释处理

**行注释**（`//`）：`lineComment()` 消费直到 `\n` 或文件末尾的所有字符，不产生 Token。

**块注释**（`/* ... */`）：`blockComment()` 消费直到 `*/` 的所有字符（遇到 `\n` 递增行号），不产生 Token。✅ **支持嵌套块注释**（depth 计数器）。

### 错误恢复

Lexer 不再产生 `INVALID` token。遇到非法字符时，调用 `error()` 输出错误信息并**跳过该字符**继续扫描：

- **孤立 `&` / `|`**（期望 `&&` / `||`）→ 报错提示，跳过
- **未闭合字符串** → 报错 `"Unterminated string"`，best-effort 发出已收集内容为 STRING token
- **其他非法字符**（`@`、`#`、`$` 等）→ 报错 `"Unexpected character: 'X'"`，跳过

```cpp
void Lexer::error(const std::string& message) {
    std::cerr << "[line " << line << "] Lexer error: " << message << "\n";
}
```

这使得 Parser 的 `synchronize()` 机制可以正常处理语法错误，而不会被 `INVALID` token 阻断。

## 扩展指南

### 添加新的运算符

1. 在 `token.h` 的 `TokenType` 枚举中添加新类型
2. 在 `token.cpp` 的 `tokenTypeToString()` 中添加对应 case
3. 在 `lexer.cpp` 的 `scanToken()` 中添加字符分发逻辑
4. 如需在 `keywords` 表中添加，更新 `lexer.cpp` 中的静态初始化

### 添加新的关键字

1. 在 `TokenType` 枚举中添加
2. 在 `tokenTypeToString()` 中添加
3. 在 `Lexer::keywords` 静态表中添加 `{"keyword", TokenType::KEYWORD}`
