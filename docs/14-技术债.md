### 1.2 🟠 中等（8 个）→ ✅ 全部修复

#### ~~BUG-002：`push()` 无栈溢出保护~~ ✅ 已修复

- **修复**：添加 `MAX_STACK_SIZE` 常量（65536），`push()` 中检查 `stackTopIndex >= MAX_STACK_SIZE` 时触发运行时错误。

#### ~~BUG-003：`pop()` 无下溢保护~~ ✅ 已修复

- **修复**：添加 `assert(stackTopIndex > 0)` + 运行时检查。

#### ~~BUG-004：编译期常量折叠整数溢出（未定义行为）~~ ✅ 已修复

- **修复**：使用可移植溢出检查（`__builtin_add_overflow` 在 GCC/Clang，手动检查在 MSVC），溢出时回退到运行时提升为 double。

#### ~~BUG-005：`continue` 在 for-in 循环中泄漏栈值~~ ✅ 已修复

- **修复**：将 `extraLocalsToPop` 拆分为 `extraLocalsToPopOnBreak` / `extraLocalsToPopOnContinue`，`visitContinueStmt` 现在正确弹出 for-in 生成的迭代器局部变量。

#### ~~BUG-006：全局变量槽位静默截断为 uint8_t~~ ✅ 已修复

- **修复**：新增 `OP_GET_GLOBAL_WIDE`、`OP_SET_GLOBAL_WIDE`、`OP_DEFINE_GLOBAL_WIDE` 指令支持 16 位全局槽位（最大 65536 个全局变量）。新增 `emitGetGlobal`/`emitSetGlobal`/`emitDefineGlobal` 辅助方法自动选择 8 位或 16 位编码。

#### ~~BUG-007：`runtimeError` 对 `currentChunk` 无空指针检查~~ ✅ 已修复

- **修复**：添加 `currentChunk` 空指针守卫，回退到无位置信息的错误消息。

#### ~~BUG-008：`decodeLineFromChunk` 越界时静默返回 0,0~~ ✅ 已修复

- **修复**：越界时返回哨兵值并打印警告，确保异常展开路径上错误消息显示正确位置。

#### ~~BUG-009：`ClassConstructor` 每次构造都快照全部全局变量~~ ✅ 已修复

- **修复**：改为内联执行（`ClassConstructor::call(vm, args)`），使用 `copyGlobalsFrom`/`mergeGlobalsTo` 在调用者 VM 上直接操作，增量脏标记（`markGlobalDirty`/`clearDirtyGlobals`）实现 O(dirty) 合并。

### 8.2 🔴 P0：`assert()` 调用 — Debug 构建中直接终止

`assert()` 在 Debug 构建中调用 `abort()`，产生 SIGABRT 信号，进程直接终止。这不是 RuntimeError，用户看到的是 "Assertion failed" 消息和非零退出码。

#### CRASH-002：编译器 locals 栈状态断言（4 处）✅ 已修复

- **文件**：`src/vm/compiler_expr.cpp`
- **修复**：4 个 `assert()` 替换为 `if` 检查 + `error("Internal compiler error: ...")` + `return`，在 Debug 和 Release 构建中一致报告编译错误。

#### CRASH-005：`builtins.cpp` range 参数索引 ✅ 已验证安全
