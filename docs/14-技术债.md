## 一、Vora 核心 (`D:\Vora\src`)

### 1.1 🔴 严重（1 个）

#### BUG-001：OP_TAIL_CALL 回退路径对 BoundMethod 错误

- **文件**：`src/vm/vm.cpp:1271-1312`
- **描述**：`OP_TAIL_CALL` 有两个路径：优化路径（复用 VoraFunction）和回退路径（普通调用 + return）。回退路径中，`callValue(callee, argCount)` 调用后立即执行 `pop()`（第 1278 行），但 `callValue` 对 `BoundMethod` 是**异步**的——它设置调用帧后返回 `true`，方法体尚未执行。此时 `pop()` 弹出的是方法参数而非返回值。
- **影响**：对绑定方法的尾调用（如 `return obj.method(args)`）会产生错误结果。
- **修复建议**：回退路径不应在 `callValue` 后立即 pop，而是等待方法体执行完毕。

### 1.2 🟠 中等（8 个）

#### BUG-002：`push()` 无栈溢出保护

- **文件**：`src/vm/vm.cpp:131-137`
- **描述**：`VM::push()` 无边界检查。虽然栈预分配 65536 个槽位，但深层嵌套表达式或大型字面量可超出此限制。`std::vector::push_back` 会重新分配，而 `stackTopIndex` 溢出将导致内存损坏。
- **修复建议**：添加 `MAX_STACK_SIZE` 常量和溢出检查。

#### BUG-003：`pop()` 无下溢保护

- **文件**：`src/vm/vm.cpp:139-141`
- **描述**：`VM::pop()` 无条件递减 `stackTopIndex`（`size_t`）。栈为空时调用会回绕到 `SIZE_MAX`，导致越界访问。
- **修复建议**：添加断言或运行时检查。

#### BUG-004：编译期常量折叠整数溢出（未定义行为）

- **文件**：`src/vm/compiler_expr.cpp:203-213`
- **描述**：`int64_t` 运算（`+`、`-`、`*`）的编译期折叠不检查溢出。C++ 中有符号整数溢出是未定义行为。运行时路径会提升为 `double`（明确定义），但编译期路径保持 `int64_t`。
- **修复建议**：编译期折叠时检查溢出，溢出时回退到运行时路径或报错。

#### BUG-005：`continue` 在 for-in 循环中泄漏栈值

- **文件**：`src/vm/compiler_stmt.cpp:1019-1053`
- **描述**：`visitBreakStmt` 正确添加了 `loopStack.back().extraLocalsToPop`（第 1007 行），但 `visitContinueStmt` **没有**（第 1031-1043 行）。for-in 循环生成的 `_iter`、`_i`、`_len` 局部变量（extraLocalsToPop=3）在 `continue` 时不会被弹出，每次 `continue` 泄漏 3 个栈值。
- **修复建议**：`visitContinueStmt` 中添加 `extraLocalsToPop` 处理。

#### BUG-006：全局变量槽位静默截断为 uint8_t

- **文件**：`src/vm/compiler_expr.cpp:120-121`
- **描述**：`resolveGlobal(base)` 返回 `int`，但 `emitByte(static_cast<uint8_t>(slot))` 截断为 8 位。超过 256 个全局变量时，槽位静默回绕，访问错误的全局变量。
- **修复建议**：添加溢出检查，超过 256 时报告编译错误。

#### BUG-007：`runtimeError` 对 `currentChunk` 无空指针检查

- **文件**：`src/vm/vm.cpp:195`
- **描述**：`runtimeError(const std::string&)` 计算 `ip - currentChunk->code.data()` 时不检查 `currentChunk` 是否为空。模块解析失败等场景可能触发空指针解引用。
- **修复建议**：添加空指针检查。

#### BUG-008：`decodeLineFromChunk` 越界时静默返回 0,0

- **文件**：`src/vm/vm.cpp:167-190`
- **描述**：如果字节码偏移超出 RLE `lines`/`columns` 向量的总长度，函数返回 `line=0, column=0` 而无警告。异常展开到无效处理器时可能发生，导致运行时错误消息显示错误的源码位置。
- **修复建议**：返回一个哨兵值或打印警告。

#### BUG-009：`ClassConstructor` 每次构造都快照全部全局变量

- **文件**：`src/runtime/class_constructor.cpp:31-32`
- **描述**：每次实例化类时，`ClassConstructor::call()` 创建临时 VM 并复制整个全局状态（`globalNames_`、`globalValues_`、`globalDefined_`、`globalIndex_`）。对频繁构造且全局变量较多的程序，开销显著。
- **修复建议**：仅在构造函数实际访问全局变量时才快照，或缓存快照。

## 二、Vora-LSP (`D:\Vora-LSP`)

### 2.1 🟠 中等（9 个）

#### LSP-001：VS Code 扩展配置键路径错误

- **文件**：`D:\Vora-LSP\src\extension.ts:29`
- **描述**：`config.get('vora.lsp.serverPath', 'vora-lsp')` — `getConfiguration('vora')` 已作用域到 `vora` 命名空间，正确路径应为 `lsp.serverPath`。当前代码使用户的服务器路径配置**静默被忽略**。
- **修复建议**：改为 `config.get<string>('lsp.serverPath', 'vora-lsp')`。

#### LSP-002：格式化返回错误的响应格式

- **文件**：`D:\Vora-LSP\server\lsp_server.cpp:623-626,639-642`
- **描述**：`handleFormatting` 返回 `{"error": "Document not found"}` 等 JSON 对象。`textDocument/formatting` 应返回 `TextEdit[]` 数组。返回 JSON 对象会被 LSP 客户端误解为有效的编辑数组，导致静默失败。
- **修复建议**：返回空数组 `nlohmann::json::array()`。

#### LSP-003：语法高亮缺少运算符

- **文件**：`D:\Vora-LSP\syntaxes\vora.tmLanguage.json:154-188`
- **描述**：以下运算符无高亮规则：`??`（空值合并）、`?.`（可选链）、`=>`（箭头函数）、`..`（范围）、`..=`（范围赋值）。它们会被解析为单独的标点符号。
- **修复建议**：添加对应的正则匹配规则。

#### LSP-004：`package.json` 引用不存在的图标

- **文件**：`D:\Vora-LSP\package.json:8`
- **描述**：`"icon": "vora_logo.png"` — 仓库中只有 `vora_logo.svg`，无 PNG 文件。扩展在 VS Code 市场和扩展列表中将缺少图标。
- **修复建议**：从 SVG 生成 PNG，或更新引用为 SVG。

#### LSP-005：每次 didOpen/didChange 触发两次语义分析

- **文件**：`D:\Vora-LSP\server\lsp_server.cpp:803-806`
- **描述**：`publishDiagnostics` 创建新的 `SemanticAnalyzer` 并丢弃结果。后续任何语言功能请求又创建并缓存一个。每次文档变更触发两次分析，浪费资源。
- **修复建议**：`publishDiagnostics` 调用 `parseAndAnalyze(uri)` 复用缓存。

#### LSP-006：`analyzeImportedFile` 路径拼接错误

- **文件**：`D:\Vora-LSP\server\lsp_server.cpp:988-993`
- **描述**：当 `std::ifstream file(filePath)` 失败时，尝试 `workspaceRoot_ + "/std/" + filePath`。但 `filePath` 此时已是绝对路径，拼接结果类似 `/workspace/std/C:/Users/file.va`，永远不匹配。
- **修复建议**：`std/` 回退逻辑应在 `resolveImportPath` 中处理。

#### LSP-007：构建系统硬编码 Debug 路径

- **文件**：`D:\Vora-LSP\CMakeLists.txt:12-13,23`
- **描述**：`VORA_BUILD` 默认为 `windows-x64-debug`，`find_library` 仅搜索 `Debug` 子目录。Release 构建时找不到库。
- **修复建议**：支持 Debug 和 Release 配置，或自动检测。

#### LSP-008：构建系统 post-build 使用 `.exe` 扩展名

- **文件**：`D:\Vora-LSP\CMakeLists.txt:67`
- **描述**：`copy_if_different ... vora-lsp.exe` 硬编码 `.exe`。Linux/macOS 上二进制文件无此扩展名，post-build 复制会失败。
- **修复建议**：使用 `$<TARGET_FILE:vora-lsp>` 或条件扩展名。

#### LSP-009：补全缺少 Set/Map 方法

- **文件**：`D:\Vora-LSP\server\lsp_server.cpp:270-305`
- **描述**：输入 `.` 后，服务器建议 Array、String、Dict 方法，但**不包含** Set 和 Map 方法（`add`、`has`、`remove`、`clear`、`values`、`keys` 等）。
- **修复建议**：添加 Set 和 Map 方法到补全列表。
