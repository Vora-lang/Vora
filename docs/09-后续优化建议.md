# 后续优化与开发建议

## 一、语言特性补全（短期优先）

### 1. break / continue

循环控制流是基本需求。

**建议实现**：
- 定义 `BreakSignal` 和 `ContinueSignal` 异常类（类似 `ReturnSignal`）
- 在 `WhileStmt` / `ForStmt` 的执行循环中 `try-catch`
- 添加 Token 类型 `BREAK`、`CONTINUE`
- Parser 中处理为新的语句类型

```
相关文件：token.h, lexer.cpp, stmt.h, parser.cpp, interpreter.cpp
```

### 2. 字符串运算符重载

当前 `+` 仅支持数字。应支持字符串拼接：

```vora
let greeting = "Hello" + " " + "World"
```

**建议实现**：在 `Interpreter::evaluate()` 的 `BinaryExpr` 分支中，增加 `string + string` 的处理。

### 3. 数组内建方法

```vora
let arr = [1, 2, 3]
arr.push(4)          // 追加
arr.pop()            // 弹出末尾
arr.length()         // 长度
```

**建议实现方案**：
- 方案 A：在 `PropertyExpr` 求值时，对 `shared_ptr<Array>` 特殊处理，识别内建方法名
- 方案 B：将 Array 包装为 ObjectInstance，注册内建方法

### 4. 字符串内建方法

```vora
let s = "hello"
s.length()           // 5
s.substring(0, 3)    // "hel"
s.contains("ell")    // TRUE
```

### 5. 更多内建函数

| 函数 | 说明 |
|------|------|
| `type(value)` | 返回值的类型名字符串 |
| `toString(value)` | 值转字符串 |
| `toNumber(string)` | 字符串转数字 |
| `len(array_or_string)` | 长度 |
| `range(n)` | 生成 `[0, 1, ..., n-1]` 数组 |

### 6. 三元运算符

```vora
let result = x > 0 ? "positive" : "negative"
```

### 7. 增强赋值运算符

```vora
x += 1
x -= 1
x *= 2
```

### 8. else if 语法糖

当前已支持 `else if`（通过嵌套 IfStmt），但可以考虑作为独立语法。

## 二、错误处理系统（中期）

### 1. try / catch / finally

```vora
try {
    let x = 1 / 0
} catch (e) {
    print("Error: ${e}")
} finally {
    print("Done")
}
```

**实现要点**：
- 新增 `TryStmt`、`CatchStmt`、`FinallyStmt` AST 节点
- 将 `RuntimeError` 暴露为 Vora 可捕获的对象
- `finally` 块始终执行

### 2. 自定义错误

```vora
Obj ValidationError(msg) {
    this.message = msg
}

throw ValidationError("Invalid input")
```

### 3. 改进错误信息

当前错误信息较为简单，建议：
- 输出出错的源代码行和列号
- 标记出错位置（类似 GCC 的 `^` 指示符）
- 支持调用栈追踪（stack trace）

## 三、模块系统（中期）

### 1. import / export

```vora
// math.va
export func add(a, b) { return a + b }
export let PI = 3.14159

// main.va
import "math"
print(math.add(1, 2))
print(math.PI)
```

**实现要点**：
- 文件系统路径解析
- 模块缓存（避免重复加载）
- 导出符号的命名空间隔离

### 2. 标准库

利用模块系统填充 `std/` 目录：

| 模块 | 功能 |
|------|------|
| `std/math` | `sqrt`, `abs`, `floor`, `ceil`, `sin`, `cos`, `random` |
| `std/string` | `split`, `join`, `trim`, `upper`, `lower` |
| `std/io` | `readFile`, `writeFile`, `readLine` |
| `std/os` | `args`, `exit`, `env` |

## 五、性能优化（长期）

### 1. 从 AST 解释器到字节码 VM（第二版）

当前的树遍历解释器每次执行都要遍历 AST，效率较低。建议：

**字节码编译器**：
```
AST → 字节码指令序列 → VM 执行
```

**优势**：
- 消除 `dynamic_cast` 开销
- 指令分发使用 switch/跳转表
- 更紧凑的内存表示
- 为后续 JIT 打基础

**核心组件**：
| 组件 | 说明 |
|------|------|
| `Chunk` | 字节码指令序列 + 常量池 |
| `OpCode` | 指令枚举（`OP_ADD`, `OP_LOAD_CONST`, `OP_CALL` 等） |
| `Compiler` | AST → 字节码编译 |
| `VM` | 栈式虚拟机，执行字节码 |

### 2. Interned Strings（字符串驻留）

当前变量名使用 `std::string` 比较。可以使用字符串驻留（interning）：
- 所有标识符名转为唯一整数 ID
- 变量查找从字符串哈希比较变为整数比较

### 3. 值表示优化

当前 `std::variant` 的 `Value` 类型：
- 每次访问有 variant 访问开销
- `shared_ptr` 堆分配开销

可考虑 **NaN Boxing** 技术：将所有值类型编码在 64 位 double 的 NaN 空间中。

### 4. 作用域查找优化

当前 `Environment::get()` 沿链表线性查找。可以：
- 在解析阶段计算每个变量的作用域深度和索引
- 运行时通过索引直接访问（类似 Lua 的 upvalue）

## 六、开发体验优化（持续）

### 1. 错误恢复

Parser 目前遇到错误立即终止。建议实现 **panic mode**：
- 遇到错误时跳过 Token 直到找到同步点（`;`、`}`）
- 继续解析后续代码，**收集多个错误一次性报告**

### 2. LSP 支持

为 Vora 实现 Language Server Protocol：
- 语法高亮
- 自动补全
- 跳转定义
- 悬停提示

### 3. 代码格式化工具

实现 `vora fmt` 命令，自动格式化 `.va` 文件。

### 4. 跨平台支持

当前仅支持 Windows（PowerShell 构建脚本）。建议：
- 添加 CMake 预设（CMakePresets.json）
- 支持 Linux / macOS 构建
- GitHub Actions CI 自动化构建和测试

### 5. 包管理器 vpm

README 中提到的 `vpm` 包管理器：
- 包注册与发现
- 依赖解析
- 版本管理
