# 后续优化与开发建议

## 一、语言特性补全（短期优先）

### 1. break / continue ✅ 已完成

循环控制流已实现。

**实现方案**：
- ✅ 定义 `BreakSignal` 和 `ContinueSignal` 异常类（类似 `ReturnSignal`）
- ✅ 在 `WhileStmt` / `ForStmt` 的执行循环中 `try-catch`
- ✅ 添加 Token 类型 `BREAK`、`CONTINUE`
- ✅ Parser 中处理为新的语句类型

### 2. 字符串运算符重载 ✅ 已完成

```vora
let greeting = "Hello" + " " + "World"  // ✅ 已支持
```

`+` 运算符现支持 `string + string` 拼接。`+=` 同理。

### 3. 数组内建方法

```vora
let arr = [1, 2, 3]
arr.push(4)          // 追加
arr.pop()            // 弹出末尾
arr.length()         // 长度
```

**建议实现方案**：
- 方案 A：在 `PropertyExpr` 求值时，对 `shared_ptr<Array>` 特殊处理，识别内建方法名
- 方案 B：将 Array 包装为 ObjectInstance，注册内建方法

### 4. 字符串内建方法

```vora
let s = "hello"
s.length()           // 5
s.substring(0, 3)    // "hel"
s.contains("ell")    // TRUE
```

### 5. 更多内建函数

| 函数 | 说明 | 状态 |
|------|------|------|
| `type(value)` | 返回值的类型名字符串 | 待实现 |
| `toString(value)` | 值转字符串 | 待实现 |
| `toNumber(string)` | 字符串转数字 | ✅ `int()`/`float()` 已覆盖 |
| `len(array_or_string)` | 长度 | 待实现 |
| `range(n)` | 生成 `[0, 1, ..., n-1]` 数组 | ✅ 已实现，支持 1/2/3 参数 |

### 6. 三元运算符 ✅ 已完成

```vora
let result = x > 0 ? "positive" : "negative"
```

**实现方案**：
- ✅ Token 类型 `QUESTION`（`?`），复用已有 `COLON`（`:`）
- ✅ AST 节点 `TernaryExpr`（condition / thenBranch / elseBranch）
- ✅ Pratt 解析器中作为中缀运算符处理，优先级 2（低于比较，高于赋值）
- ✅ 右结合（`a ? b : c ? d : e` → `a ? b : (c ? d : e)`）
- ✅ 短路求值（只求值被选中的分支）
- ✅ AST Printer 支持

### 7. 增强赋值运算符 ✅ 已完成

```vora
x += 1    // ✅
x -= 1    // ✅
x *= 2    // ✅
x /= 2    // ✅
x %= 2    // ✅
```

支持数字运算、字符串拼接 (`s += "!"`)、数组追加 (`arr += [3, 4]`)。

### 8. else if 语法糖

当前已支持 `else if`（通过嵌套 IfStmt），但可以考虑作为独立语法。

## 二、错误处理系统（中期）

### 1. try / catch / finally ✅ 已完成

```vora
try {
    let x = 1 / 0
} catch (e) {
    print("Error: ${e}")
} finally {
    print("Done")
}
```

**实现方案**：
- ✅ AST 节点 `TryStmt`（tryBlock / catchVar / catchBlock / finallyBlock，catch 和 finally 均可选）
- ✅ `finally` 块始终执行——捕获所有控制流信号（ReturnSignal / BreakSignal / ContinueSignal / ThrowSignal），执行 finally 后重新抛出
- ✅ `catch (e)` 捕获内建 `RuntimeError` 时，`e` 绑定为错误消息字符串
- ✅ 支持 `try/catch`、`try/finally`、`try/catch/finally` 三种组合
- ✅ 修复 `executeBlock` / `callFunction` 的作用域泄漏 bug（catch-all → popScope → rethrow）

### 2. 自定义错误 ✅ 已完成

```vora
Obj ValidationError(msg) {
    this.message = msg
}

throw ValidationError("Invalid input")
```

**实现方案**：
- ✅ 新增 `ThrowSignal{Value}` 异常类型，携带任意 Vora 值
- ✅ `throw <expr>` 语句：求值表达式并以 `ThrowSignal` 抛出
- ✅ `catch (e)` 同时捕获 `ThrowSignal`（绑定原值）和 `RuntimeError`（绑定为字符串）
- ✅ 自定义错误对象：通过 `Obj` 定义，`throw` 抛出实例，`catch` 中访问 `.message` 等属性

### 3. 改进错误信息

当前错误信息较为简单，建议：
- 输出出错的源代码行和列号
- 标记出错位置（类似 GCC 的 `^` 指示符）
- 支持调用栈追踪（stack trace）

## 五、性能优化（长期）

### 1. 从 AST 解释器到字节码 VM（第二版）

当前的树遍历解释器每次执行都要遍历 AST，效率较低。建议：

**字节码编译器**：
```
AST → 字节码指令序列 → VM 执行
```

**优势**：
- 消除 `dynamic_cast` 开销
- 指令分发使用 switch/跳转表
- 更紧凑的内存表示
- 为后续 JIT 打基础

**核心组件**：
| 组件 | 说明 |
|------|------|
| `Chunk` | 字节码指令序列 + 常量池 |
| `OpCode` | 指令枚举（`OP_ADD`, `OP_LOAD_CONST`, `OP_CALL` 等） |
| `Compiler` | AST → 字节码编译 |
| `VM` | 栈式虚拟机，执行字节码 |

### 2. Interned Strings（字符串驻留）

当前变量名使用 `std::string` 比较。可以使用字符串驻留（interning）：
- 所有标识符名转为唯一整数 ID
- 变量查找从字符串哈希比较变为整数比较

### 3. 值表示优化

当前 `std::variant` 的 `Value` 类型：
- 每次访问有 variant 访问开销
- `shared_ptr` 堆分配开销

可考虑 **NaN Boxing** 技术：将所有值类型编码在 64 位 double 的 NaN 空间中。

### 4. 作用域查找优化

当前 `Environment::get()` 沿链表线性查找。可以：
- 在解析阶段计算每个变量的作用域深度和索引
- 运行时通过索引直接访问（类似 Lua 的 upvalue）

## 六、开发体验优化（持续）

### 1. 错误恢复

Parser 目前遇到错误立即终止。建议实现 **panic mode**：
- 遇到错误时跳过 Token 直到找到同步点（`;`、`}`）
- 继续解析后续代码，**收集多个错误一次性报告**

### 2. LSP 支持

为 Vora 实现 Language Server Protocol：
- 语法高亮
- 自动补全
- 跳转定义
- 悬停提示

### 3. 代码格式化工具

实现 `vora fmt` 命令，自动格式化 `.va` 文件。

### 4. 跨平台支持

当前仅支持 Windows（PowerShell 构建脚本）。建议：
- 添加 CMake 预设（CMakePresets.json）
- 支持 Linux / macOS 构建
- GitHub Actions CI 自动化构建和测试

### 5. 包管理器 vpm

README 中提到的 `vpm` 包管理器：
- 包注册与发现
- 依赖解析
- 版本管理
