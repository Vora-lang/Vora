# 后续优化与开发建议

## 一、语言特性补全（短期优先）

### 1. break / continue ✅ 已完成

循环控制流已实现。

**实现方案**：
- ✅ 定义 `BreakSignal` 和 `ContinueSignal` 异常类（类似 `ReturnSignal`）
- ✅ 在 `WhileStmt` / `ForStmt` 的执行循环中 `try-catch`
- ✅ 添加 Token 类型 `BREAK`、`CONTINUE`
- ✅ Parser 中处理为新的语句类型

### 2. 字符串运算符重载 ✅ 已完成

```vora
let greeting = "Hello" + " " + "World"  // ✅ 已支持
```

`+` 运算符现支持 `string + string` 拼接。`+=` 同理。

### 3. 数组内建方法 ✅ 已完成（v0.12）

```vora
let arr = [1, 2, 3]
arr.add(4)           // 追加
arr.pop()            // 弹出末尾
arr.length           // 长度（属性）
arr.insert(0, "x")   // 插入
arr.remove(1)        // 删除
arr.indexOf(2)       // 查找
arr.clear()          // 清空
```

**实现方案**：方案 A（属性拦截）——在 `PropertyExpr` 求值和 VM `OP_GET_PROPERTY` 中，对 `shared_ptr<Array>` 特殊处理，识别 7 个内建方法。

### 4. 字符串内建方法 ✅ 已完成（v0.12）

```vora
let s = "hello"
s.length             // 5
s.substring(0, 3)    // "hel"

s.upper()            // "HELLO"
s.replace("l", "L")  // "heLlo"

```

### 5. 更多内建函数

| 函数 | 说明 | 状态 |
|------|------|------|
| `type(value)` | 返回值的类型名字符串（区分 int/float） | ✅ 已实现 |
| `toString(value)` | 值转字符串 | 待实现（`str()` 已覆盖部分场景） |

### 6. 三元运算符 ✅ 已完成

```vora
let result = x > 0 ? "positive" : "negative"
```

**实现方案**：
- ✅ Token 类型 `QUESTION`（`?`），复用已有 `COLON`（`:`）
- ✅ AST 节点 `TernaryExpr`（condition / thenBranch / elseBranch）
- ✅ Pratt 解析器中作为中缀运算符处理，优先级 2（低于比较，高于赋值）
- ✅ 右结合（`a ? b : c ? d : e` → `a ? b : (c ? d : e)`）
- ✅ 短路求值（只求值被选中的分支）
- ✅ AST Printer 支持

### 7. 增强赋值运算符 ✅ 已完成

```vora
x += 1    // ✅
x -= 1    // ✅
x *= 2    // ✅
x /= 2    // ✅
x %= 2    // ✅
```

支持数字运算、字符串拼接 (`s += "!"`)、数组追加 (`arr += [3, 4]`)。

### 8. else if 语法糖

当前已支持 `else if`（通过嵌套 IfStmt），但可以考虑作为独立语法。

## 二、错误处理系统（中期）

### 1. try / catch / finally ✅ 已完成

```vora
try {
    let x = 1 / 0
} catch (e) {
    print("Error: ${e}")
} finally {
    print("Done")
}
```

**实现方案**：
- ✅ AST 节点 `TryStmt`（tryBlock / catchVar / catchBlock / finallyBlock，catch 和 finally 均可选）
- ✅ `finally` 块始终执行——捕获所有控制流信号（ReturnSignal / BreakSignal / ContinueSignal / ThrowSignal），执行 finally 后重新抛出
- ✅ `catch (e)` 捕获内建 `RuntimeError` 时，`e` 绑定为错误消息字符串
- ✅ 支持 `try/catch`、`try/finally`、`try/catch/finally` 三种组合
- ✅ 修复 `executeBlock` / `callFunction` 的作用域泄漏 bug（catch-all → popScope → rethrow）

### 2. 自定义错误 ✅ 已完成

```vora
Obj ValidationError(msg) {
    this.message = msg
}

throw ValidationError("Invalid input")
```

**实现方案**：
- ✅ 新增 `ThrowSignal{Value}` 异常类型，携带任意 Vora 值
- ✅ `throw <expr>` 语句：求值表达式并以 `ThrowSignal` 抛出
- ✅ `catch (e)` 同时捕获 `ThrowSignal`（绑定原值）和 `RuntimeError`（绑定为字符串）
- ✅ 自定义错误对象：通过 `Obj` 定义，`throw` 抛出实例，`catch` 中访问 `.message` 等属性

### 3. 改进错误信息

当前错误信息较为简单，建议：
- 输出出错的源代码行和列号
- 标记出错位置（类似 GCC 的 `^` 指示符）
- 支持调用栈追踪（stack trace）

## 五、性能优化（长期）

### 1. 从 AST 解释器到字节码 VM（第二版）✅ Phase 1-4 + v0.10 异常修复 已完成

字节码 VM 已完全实现并作为默认执行后端。

**Phase 1**：表达式、全局变量、if/else、while、三元运算符、数组、索引、内建函数调用。
**Phase 2**：局部变量、块作用域、break/continue、for-in 循环、函数声明与调用、基本 try/catch/throw。
**Phase 3**：对象（Obj）、属性读写、方法调用、继承（含多层继承）。
**Phase 4**：闭包（upvalue 机制）、字符串插值（编译时解析）、运行时错误 catch 完善。
**v0.10 修复**：finally 路由（break/continue/return 穿透 finally 块）、OP_FINALLY_END 异常重抛、catch handler 清理、for-in break 栈泄漏修复。

**已实现组件**：
| 组件 | 文件 | 说明 |
|------|------|------|
| `OpCode` | `src/vm/opcode.h` | 50 条指令枚举（含 OP_PUSH_CATCH/OP_POP_CATCH/OP_CLEAR_EXCEPTION/OP_THROW/OP_FINALLY_END） |
| `Chunk` | `src/vm/chunk.h/.cpp` | 字节码 + 常量池 + RLE 行/列号 + 反汇编器 |
| `Compiler` | `src/vm/compiler.h/.cpp` | AST → 字节码，含局部变量跟踪、循环上下文、函数/对象编译、upvalue 解析、try/finally 路由 |
| `VM` | `src/vm/vm.h/.cpp` | 栈式虚拟机（1024 槽位 + 调用帧 + 异常处理器 + exceptionInFlight + openUpvalues） |

**已完成**：闭包、字符串插值、异常处理全覆盖（含 finally 路由 + 异常重抛）。

### 2. Interned Strings（字符串驻留）✅ 全局变量已完成

~~当前变量名使用 `std::string` 比较。可以使用字符串驻留（interning）：~~

- ✅ **全局变量已完成**：编译器对全局变量名进行驻留（`globalNames` 向量 + `resolveGlobal()`），VM 端使用整数槽位（`OP_GET_GLOBAL slot`）替代字符串常量池查找，全局访问从 `unordered_map` 哈希查找变为 O(1) 数组索引。
- ⏳ **局部变量**：当前通过编译器 `locals` 向量按索引定位，已无需字符串比较（`OP_GET_LOCAL slot`）。
- ⏳ **对象属性**：仍使用 `std::map<string, Value>` 字符串键查找，可考虑驻留优化。

### 3. 值表示优化

当前 `std::variant` 的 `Value` 类型：
- 每次访问有 variant 访问开销
- `shared_ptr` 堆分配开销

可考虑 **NaN Boxing** 技术：将所有值类型编码在 64 位 double 的 NaN 空间中。

### 4. 作用域查找优化

~~当前 `Environment::get()` 沿链表线性查找。~~ ✅ 解释器局部变量已通过编译器 `resolveLocal()` + `OP_GET_LOCAL` 实现索引直访。全局变量已通过驻留数组实现 O(1) 访问。

可进一步优化：
- ⏳ 在解析阶段计算每个变量的作用域深度和索引
- ⏳ 运行时通过索引直接访问（类似 Lua 的 upvalue）——VM 已实现 upvalue 机制
