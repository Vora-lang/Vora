## 各文件详细说明

### 入口文件

#### `src/main.cpp`（~313 行）

程序入口点，负责：

- **命令行参数解析**：支持 `--repl`、`--ast-printer`、`--tokens`、`fmt`、`-w`/`--write`、文件路径
- **文件读取**：`readFile()` 读取 `.va` 源文件，并去除 UTF-8 BOM 头
- **脚本执行**：`runScript()` 串联 Lexer → Parser → Compiler → VM
- **REPL 模式**：`runREPL()` 实现交互式读取-求值-打印循环
- **格式化模式**：`runFmt()` 执行 `vora fmt`，支持格式化到 stdout 或原地写回（`-w`）
- **全局初始化**：VM 模式下通过 `vm.initGlobals()` 预分配编译器驻留的全局变量槽位，并注册 12 个内建函数
- **错误处理**：捕获 `RuntimeError` 异常并输出行号/列号和错误信息（含源码行+插入符标记）

**优先级表**：

| 优先级 | 运算符 | 结合性 |
|--------|--------|--------|
| 1 | `=`, `||` | 右结合（`=`） |

| 4 | `<`, `<=`, `>`, `>=` | 左结合 |
| 5 | `+`, `-` | 左结合 |
| 6 | `*`, `/`, `%` | 左结合 |
| 7 | `**` | 右结合 |

#### `src/parser/parser.cpp`（1114 行）

Parser 的完整实现。表达式解析使用 Pratt 算法，语句解析使用递归下降。赋值（`=`）在 Pratt 中统一处理，通过 `dynamic_cast` 判断左侧是 `VariableExpr` 还是 `PropertyExpr`，分别生成 `AssignmentExpr` 或 `PropertyAssignmentExpr`。支持 `break`/`continue`/`throw`/`try`、三元运算符 `? :`、自增/自减、字符串插值等。

### 运行时系统（Runtime）

#### `src/runtime/value.h`（95 行）

定义 Vora 运行时值类型系统。

**类型别名 `Value`**（11 种类型）：
```cpp
using Value = std::variant<
    std::nullptr_t,                          // null
    double,                                  // float64 浮点数
    int64_t,                                 // int64 整数
    bool,                                    // 布尔（true / false）
    std::string,                             // 字符串
    GcPtr<Array>,                            // 数组（GC 管理）
    GcPtr<Dict>,                             // 字典（GC 管理）
    GcPtr<Callable>,                         // 可调用对象（GC 管理）
    GcPtr<ObjectInstance>,                   // 对象实例（GC 管理）
    GcPtr<FunctionPrototype>,                // 编译后的函数原型（VM 常量池用）
    GcPtr<ClassDefinition>                   // 编译后的类定义（VM 常量池用，统一编译期+运行时）
```
