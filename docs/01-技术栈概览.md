# 技术栈概览

## 项目简介

Vora 是一门自定义编程语言，使用 **字节码编译器+栈式虚拟机** 作为唯一的执行后端。项目灵感来源于 *Crafting Interpreters*、Lua、Wren 和 CPython。

**Vora is a dynamically typed scripting language. It features JavaScript-like syntax, Lua-level simplicity, and Wren-style object orientation.**

- **语言名称**：`Vora`
- **源文件扩展名**：`.va`
- **解释器名称**：`vora`
- **执行后端**：`Vora VM`（Phase 1-4 + v0.15 全特性 — 支持表达式、控制流、局部变量、函数、闭包、对象、多继承 C3 线性化、super、异常处理、字符串插值、int64/float64 双数值、默认参数、数组/字符串方法；含全局变量驻留、快速数值操作码、常量折叠、移动语义、属性拦截、finally 路由、运行时调用栈回溯）
- **包管理器（规划中）**：`vpm`

> **v0.13 变更**：树遍历 AST 解释器已移除。VM 是唯一的执行后端。

## 核心技术栈

| 类别 | 技术 | 版本/说明 |
|------|------|-----------|
| 实现语言 | C++ | C++17 |
| 构建系统 | CMake | >= 3.16 |
| 构建脚本 | `build.ps1` (Windows) / `build.sh` (Linux/macOS) | 一键构建 + 运行 + 打包 |
| CMake 预设 | `CMakePresets.json` | 20 个预设 (x64/x86/arm64/universal × Debug/Release) |
| 打包 | CPack + WiX | `.msi` (Windows) / `.deb` `.rpm` `.pkg.tar.xz` (Linux) |
| C++ 标准 | CMAKE_CXX_STANDARD 17 | 禁用编译器扩展 |
| 外部依赖 | **无** | 纯 C++17 标准库 |
| 平台 | Windows / Linux / macOS | MSVC / GCC / Clang，支持交叉编译 |

## 关键 C++ 特性使用

| 特性 | 用途 |
|------|------|
| `std::variant` | `Value` 类型 —— 实现标签联合（tagged union），表示 Vora 中所有运行时值 |
| `std::unique_ptr` | AST 节点所有权管理，表达式和语句的唯一所有权 |
| `std::shared_ptr` | 闭包环境、可调用对象、数组、对象实例的共享所有权 |
| `std::visit` | 对 `Value` variant 进行模式匹配 |
| `dynamic_cast` | Pratt 循环中区分 `VariableExpr`/`PropertyExpr` 作为赋值目标；`objStatement()` 中区分 `FuncStmt`（方法）与其他语句（构造函数体） |
| `std::deque` | VM 编译器作用域栈（`scopeStack`）的底层容器 |
| `std::unordered_map` | VM 全局名称→槽位索引映射 |
| `std::map` | 对象属性和方法的有序存储 |

| 版本 | 方案 | 状态 |
|------|------|------|
| 第一版 | AST 树遍历解释器 | ✔ 已完成（v0.13 已移除） |
| 第二版 | 字节码虚拟机（Bytecode VM） | ✔ Phase 1-4 + v0.15 全特性（表达式/控制流/函数/闭包/对象/多继承/super/异常/插值/双数值/默认参数/方法）—— **当前唯一后端** |

## 构建与运行

### 构建（跨平台）

```bash
# Windows (PowerShell)
.\build.ps1                          # 默认 x64 Debug
.\build.ps1 -Architecture arm64 -Config Release -Package  # ARM64 + .msi

# Linux / macOS (bash)
./build.sh                           # 默认 x64 debug
./build.sh -a arm64 -c release -p    # 交叉编译 ARM64 + .deb/.rpm
```

### 支持的架构

| 平台 | 架构 |
|------|------|
| Windows | x64, x86 (Win32), ARM64 |
| Linux | x64, x86 (i386), aarch64, armhf |
| macOS | universal (x86_64 + arm64) |

### 运行

```bash
# 执行脚本文件（字节码 VM）
./build/Debug/Vora.exe examples/01-literals.va         # Windows
./build/Vora examples/01-literals.va                    # Linux/macOS

# 进入 REPL 交互模式
./build/Debug/Vora.exe --repl

# 调试：打印 AST（S-expression 格式）
./build/Debug/Vora.exe examples/01-literals.va --ast-printer

# 调试：打印词法分析 Token 流
./build/Debug/Vora.exe examples/01-literals.va --tokens

# 调试：字节码反汇编
./build/Debug/Vora.exe examples/01-literals.va --disassemble
```

## 执行流水线

```
源代码（.va 文件）
    │
    ▼
┌─────────────┐
│   Lexer      │  词法分析：源代码 → Token 流
└──────┬──────┘
       │
       ▼
┌─────────────┐
│   Parser     │  语法分析：Token 流 → AST（Program）
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  Compiler    │  AST → 字节码
└──────┬──────┘
       │
       ▼
┌─────────────┐
│     VM       │  栈式虚拟机执行字节码
└──────┬──────┘
       │
       ▼
   输出结果
```

**1. 词法分析（Lexer）**：将源代码字符串扫描为 Token 序列。

**2. 语法分析（Parser）**：使用 Pratt 解析器（优先级爬升法）将 Token 流解析为 AST。

**3. 编译（Compiler）**：Compiler（`ExprVisitor<void>` + `StmtVisitor<void>`）将 AST 编译为字节码。

**4. 执行（VM，v0.13）**：栈式 VM 执行字节码指令。支持表达式、全局/局部变量、控制流（if/while/for-in/break/continue）、函数声明与调用、闭包（upvalue 机制）、对象与继承、异常处理（try/catch/throw + finally 路由 + 异常重抛 + catch handler 清理，所有运行时错误可捕获）、字符串插值（编译时解析）、int64/float64 双数值类型、数组/字符串内建方法（属性拦截）。性能优化：全局变量整数驻留（消除运行时字符串哈希）、快速数值操作码（跳过类型检查）、常量折叠（编译期求值）、移动语义（减少 shared_ptr 引用计数开销）。
