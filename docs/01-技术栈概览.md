# 技术栈概览

## 项目简介

Vora 是一门自定义编程语言，拥有 **双后端** 执行架构：树遍历 AST 解释器（参考实现）和字节码编译器+栈式虚拟机（高性能第二后端）。项目灵感来源于 *Crafting Interpreters*、Lua、Wren 和 CPython。

- **语言名称**：`Vora`
- **源文件扩展名**：`.va`
- **解释器名称**：`vora`
- **VM 后端**：`Vora VM`（Phase 1/2/3/4 已完成 + 性能优化 — 支持表达式、控制流、局部变量、函数、闭包、对象、异常处理、字符串插值；含全局变量驻留、快速数值操作码、常量折叠、移动语义）
- **包管理器（规划中）**：`vpm`

| 类别 | 技术 | 版本/说明 |

| 实现语言 | C++ | C++17 |

## 关键 C++ 特性使用

| 特性 | 用途 |
|------|------|
| `std::variant` | `Value` 类型 —— 实现标签联合（tagged union），表示 Vora 中所有运行时值 |
| `std::unique_ptr` | AST 节点所有权管理，表达式和语句的唯一所有权 |
| `std::shared_ptr` | 闭包环境、可调用对象、数组、对象实例的共享所有权 |
| `std::visit` | 对 `Value` variant 进行模式匹配 |
| `dynamic_cast` | Pratt 循环中区分 `VariableExpr`/`PropertyExpr` 作为赋值目标 |
| `std::deque` | 作用域栈（`scopeStack`）的底层容器 |
| `std::unordered_map` | 环境（Environment）中的变量存储；VM 全局名称→槽位索引映射 |
| `std::map` | 对象属性和方法的有序存储 |

## 构建与运行

### 构建（Windows PowerShell）

```powershell
# 使用内置构建脚本（自动清理、配置、编译、运行）
.\build.ps1

# 或手动 CMake 构建
cmake -S . -B build
cmake --build build
```

### 运行

```bash
# 执行脚本文件（默认：字节码 VM）
./build/Debug/Vora.exe examples/main.va

# 执行脚本文件（树遍历解释器）
./build/Debug/Vora.exe examples/main.va --interpreter

# 进入 REPL 交互模式（仅解释器）
./build/Debug/Vora.exe --repl

# 调试：打印 AST（S-expression 格式）
./build/Debug/Vora.exe examples/main.va --ast-printer

# 调试：打印词法分析 Token 流
./build/Debug/Vora.exe examples/main.va --tokens

# 调试：字节码反汇编
./build/Debug/Vora.exe examples/main.va --tokens
```

## 执行流水线

```
源代码（.va 文件）
    │
    ▼
┌─────────────┐
│   Lexer      │  词法分析：源代码 → Token 流
└──────┬──────┘
       │
       ▼
┌─────────────┐
│   Parser     │  语法分析：Token 流 → AST（Program）
└──────┬──────┘
       │
       ├──────────────────────────────┐
       ▼                              ▼
┌─────────────┐              ┌─────────────┐
│ Interpreter  │              │  Compiler    │  AST → 字节码
│ (树遍历)     │              └──────┬──────┘
└──────┬──────┘                     ▼
       │                    ┌─────────────┐
       │                    │     VM       │  栈式虚拟机
       │                    └──────┬──────┘
       │                           │
       ▼                           ▼
   输出结果                     输出结果
```

**1. 词法分析（Lexer）**：将源代码字符串扫描为 Token 序列。

**2. 语法分析（Parser）**：使用 Pratt 解析器（优先级爬升法）将 Token 流解析为 AST。

**3a. 解释执行（Interpreter，`--interpreter`）**：树遍历解释器通过 Visitor 模式（`ExprVisitor<Value>` + `StmtVisitor<void>`）直接遍历 AST 求值。

**3b. 字节码 VM（默认）**：Compiler（`ExprVisitor<void>` + `StmtVisitor<void>`）将 AST 编译为字节码；栈式 VM 执行字节码指令。支持表达式、全局/局部变量、控制流（if/while/for-in/break/continue）、函数声明与调用、闭包（upvalue 机制）、对象与继承、异常处理（try/catch/throw，所有运行时错误可捕获）、字符串插值（编译时解析）。性能优化：全局变量整数驻留（消除运行时字符串哈希）、快速数值操作码（跳过类型检查）、常量折叠（编译期求值）、移动语义（减少 shared_ptr 引用计数开销）。
