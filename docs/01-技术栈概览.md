# 技术栈概览

## 项目简介

Vora 是一门自定义编程语言，采用 **树遍历 AST 解释器** 方案实现。项目灵感来源于 *Crafting Interpreters*、Lua、Wren 和 CPython。

- **语言名称**：`Vora`
- **源文件扩展名**：`.va`
- **解释器名称**：`vora`
- **未来 VM**：`Vora VM`
- **包管理器（规划中）**：`vpm`

## 核心技术栈

| 类别 | 技术 | 版本/说明 |
|------|------|-----------|
| 实现语言 | C++ | C++17 |
| 构建系统 | CMake | >= 3.16 |
| 构建脚本 | PowerShell (`build.ps1`) | Windows 环境一键构建 |
| C++ 标准 | CMAKE_CXX_STANDARD 17 | 禁用编译器扩展 |
| 外部依赖 | **无** | 纯 C++17 标准库 |
| 平台 | Windows (MSVC) | Visual Studio 工具链 |

## 关键 C++ 特性使用

| 特性 | 用途 |
|------|------|
| `std::variant` | `Value` 类型 —— 实现标签联合（tagged union），表示 Vora 中所有运行时值 |
| `std::unique_ptr` | AST 节点所有权管理，表达式和语句的唯一所有权 |
| `std::shared_ptr` | 闭包环境、可调用对象、数组、对象实例的共享所有权 |
| `std::visit` | 对 `Value` variant 进行模式匹配 |
| `dynamic_cast` | 运行时多态 dispatch（解释器中根据 AST 节点类型分发） |
| `std::deque` | 作用域栈（`scopeStack`）的底层容器 |
| `std::unordered_map` | 环境（Environment）中的变量存储 |
| `std::map` | 对象属性和方法的有序存储 |

## 构建与运行

### 构建（Windows PowerShell）

```powershell
# 使用内置构建脚本（自动清理、配置、编译、运行）
.\build.ps1

# 或手动 CMake 构建
cmake -S . -B build
cmake --build build
```

### 运行

```bash
# 执行脚本文件
./build/Debug/Vora.exe examples/main.va

# 进入 REPL 交互模式
./build/Debug/Vora.exe --repl

# 调试：打印 AST（S-expression 格式）
./build/Debug/Vora.exe examples/main.va --ast-printer

# 调试：打印词法分析 Token 流
./build/Debug/Vora.exe examples/main.va --tokens
```

## 执行流水线

```
源代码（.va 文件）
    │
    ▼
┌─────────────┐
│   Lexer      │  词法分析：源代码 → Token 流
└──────┬──────┘
       │
       ▼
┌─────────────┐
│   Parser     │  语法分析：Token 流 → AST（Program）
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ Interpreter  │  解释执行：遍历 AST，直接求值
└──────┬──────┘
       │
       ▼
   输出结果
```

**1. 词法分析（Lexer）**：将源代码字符串扫描为 Token 序列。

**2. 语法分析（Parser）**：使用 Pratt 解析器（优先级爬升法）将 Token 流解析为 AST。

**3. 解释执行（Interpreter）**：树遍历解释器直接遍历 AST 节点，通过 `dynamic_cast` 分发到对应的求值逻辑。
