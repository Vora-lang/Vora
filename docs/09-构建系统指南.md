# 构建系统指南

## 概述

Vora 使用 CMake 构建系统，支持以下特性：

- **多架构构建**：x86 (32-bit)、x64 (64-bit)、ARM64/aarch64、armhf (32-bit ARM)
- **跨平台**：Windows (MSVC)、Linux (GCC/Clang)、macOS (Apple Clang)
- **一键打包**：`.msi`（Windows）、`.deb`/`.rpm`/`.pkg.tar.xz`（Linux）、`.tar.gz`（macOS）
- **CMake Presets**：预配置的构建预设，无需手动指定参数

## 快速开始

### Windows (PowerShell)

```powershell
# 默认构建（x64, Debug）
.\build.ps1

# 指定架构和配置
.\build.ps1 -Architecture arm64 -Config Release

# 构建并生成安装包
.\build.ps1 -Architecture x64 -Config Release -Package
```

### Linux / macOS (bash)

```bash
# 默认构建（x64, debug）
./build.sh

# 指定架构和配置
./build.sh -a arm64 -c release

# 构建并生成安装包
./build.sh -a x64 -c release -p
```

## CMake Presets

项目使用 CMake Presets（`CMakePresets.json`）管理构建配置。命名规则：`{平台}-{架构}-{配置}`。

### 查看所有预设

```bash
cmake --list-presets
```

### 预设列表

#### Windows (MSVC — Visual Studio 2022)

| 预设 | 架构 | 配置 | 输出 |
|------|------|------|------|
| `windows-x64-debug` | x64 (64-bit) | Debug | `.exe` |
| `windows-x64-release` | x64 (64-bit) | Release | `.msi` + `.zip` |
| `windows-x86-debug` | x86 (32-bit) | Debug | `.exe` |
| `windows-x86-release` | x86 (32-bit) | Release | `.msi` + `.zip` |
| `windows-arm64-debug` | ARM64 | Debug | `.exe` |
| `windows-arm64-release` | ARM64 | Release | `.msi` + `.zip` |

#### Linux (Ninja)

| 预设 | 架构 | 配置 | 输出 |
|------|------|------|------|
| `linux-x64-debug` | x86_64 (原生) | Debug | 可执行文件 |
| `linux-x64-release` | x86_64 (原生) | Release | `.deb`/`.rpm`/`.pkg.tar.xz` |
| `linux-x86-debug` | i386 (交叉编译) | Debug | 可执行文件 |
| `linux-x86-release` | i386 (交叉编译) | Release | `.deb`/`.rpm`/`.pkg.tar.xz` |
| `linux-aarch64-debug` | aarch64 (交叉编译) | Debug | 可执行文件 |
| `linux-aarch64-release` | aarch64 (交叉编译) | Release | `.deb`/`.rpm`/`.pkg.tar.xz` |
| `linux-armhf-debug` | armv7 (交叉编译) | Debug | 可执行文件 |
| `linux-armhf-release` | armv7 (交叉编译) | Release | `.deb`/`.rpm`/`.pkg.tar.xz` |

#### macOS (Ninja)

| 预设 | 架构 | 配置 | 输出 |
|------|------|------|------|
| `macos-universal-debug` | x86_64 + arm64 | Debug | fat 二进制文件 |
| `macos-universal-release` | x86_64 + arm64 | Release | `.tar.gz` |

### 手动使用 Presets

```bash
# 1. 配置
cmake --preset linux-aarch64-release

# 2. 编译
cmake --build --preset linux-aarch64-release

# 3. 打包 (仅 Release)
cpack --preset linux-aarch64-release
# 或
cmake --build build/linux-aarch64-release --target package
```

## 交叉编译

### Linux → ARM64 (aarch64)

```bash
# 安装交叉编译工具链
sudo apt-get install gcc-aarch64-linux-gnu g++-aarch64-linux-gnu

# 构建
./build.sh -a aarch64 -c release -p
```

### Linux → ARM32 (armhf)

```bash
# 安装交叉编译工具链
sudo apt-get install gcc-arm-linux-gnueabihf g++-arm-linux-gnueabihf

# 构建
./build.sh -a armhf -c release -p
```

### Linux → x86 (32-bit)

```bash
# 安装 multilib 支持（方式一：宿主编译器 + -m32）
sudo apt-get install gcc-multilib g++-multilib

# 或安装独立交叉编译器（方式二）
sudo apt-get install gcc-i686-linux-gnu g++-i686-linux-gnu

# 构建
./build.sh -a x86 -c release -p
```

### Windows → ARM64

需要安装 Visual Studio Build Tools 的 ARM64 组件：

1. 打开 Visual Studio Installer
2. "单个组件" → 勾选 "MSVC v143 - VS 2022 C++ ARM64 生成工具"
3. 安装

```powershell
.\build.ps1 -Architecture arm64 -Config Release -Package
```

### macOS Universal Binary

无需交叉编译工具链 — Xcode 自带支持：

```bash
./build.sh -c release -p
# 生成包含 x86_64 和 arm64 的 fat binary
```

验证通用二进制：
```bash
lipo -info build/macos-universal-release/Vora
# 输出: Architectures in the fat file: ... are: x86_64 arm64
```

## 打包

### 生成的包格式

| 平台 | 格式 | 生成器 | 依赖工具 |
|------|------|--------|----------|
| Windows | `.msi` | WiX Toolset | `candle`, `light` (dotnet tool) |
| Windows | `.zip` | ZIP | 无（CMake 内置） |
| Linux | `.deb` | DEB | `dpkg` |
| Linux | `.rpm` | RPM | `rpmbuild` |
| Linux | `.pkg.tar.xz` | TXZ | 无（CMake 内置） |
| macOS | `.tar.gz` | TGZ | 无（CMake 内置） |

### 包命名规则

```
vora-{版本}-{系统}-{架构}.{扩展名}

示例:
  vora-0.12.0-Linux-amd64.deb
  vora-0.12.0-Linux-aarch64.rpm
  vora-0.12.0-Linux-armhf.tar.xz
  vora-0.12.0-Windows-amd64.msi
  vora-0.12.0-Darwin-universal.tar.gz
```

### Windows: 安装 WiX Toolset

```powershell
# 通过 dotnet 安装 WiX v4+
dotnet tool install -g wix

# 验证安装
candle --version
light --version
```

CMake 会自动检测 WiX——如果未找到，则仅生成 `.zip` 包。

## CI/CD

GitHub Actions 工作流（`.github/workflows/ci.yml`）在以下矩阵上运行：

| Runner | 架构 | 配置 |
|--------|------|------|
| `ubuntu-latest` | x64, x86, aarch64, armhf | Debug, Release |
| `windows-latest` | x64, x86, arm64 | Debug, Release |
| `macos-latest` | universal (x86_64+arm64) | Debug, Release |

每个 Release 构建都会生成对应的安装包并上传为 Artifact。

## 工具链文件

项目在 `cmake/toolchains/` 目录下提供交叉编译工具链文件：

| 文件 | 目标 | 编译器 |
|------|------|--------|
| `linux-i386.cmake` | Linux x86 (32-bit) | `i686-linux-gnu-g++` 或 `g++ -m32` |
| `linux-aarch64.cmake` | Linux ARM64 | `aarch64-linux-gnu-g++` |
| `linux-armhf.cmake` | Linux ARM32 hard-float | `arm-linux-gnueabihf-g++` |
| `windows-i386.cmake` | Windows x86 | MSVC Win32 |
| `windows-arm64.cmake` | Windows ARM64 | MSVC ARM64 |
| `macos-universal.cmake` | macOS Universal | Apple Clang (`CMAKE_OSX_ARCHITECTURES`) |

这些文件由 CMake Presets 自动引用，一般无需手动使用。

## 目录结构

```
build/
├── linux-x64-debug/Vora          # Linux x64 Debug 可执行文件
├── linux-x64-release/
│   ├── Vora                      # Linux x64 Release 可执行文件
│   ├── vora-0.12.0-Linux-amd64.deb
│   ├── vora-0.12.0-Linux-amd64.rpm
│   └── vora-0.12.0-Linux-amd64.tar.xz
├── windows-x64-debug/Debug/Vora.exe
├── windows-x64-release/
│   ├── Release/Vora.exe
│   ├── vora-0.12.0-Windows-amd64.msi
│   └── vora-0.12.0-Windows-amd64.zip
└── ...
```

每个预设使用独立的构建目录（`build/{presetName}`），互不干扰。

## 常见问题

### Q: `cmake --preset` 报 "Unknown preset"

确保在项目根目录运行命令。使用 `cmake --list-presets` 查看可用预设。

### Q: "Ninja not found" (Linux/macOS)

安装 Ninja：
```bash
# Ubuntu/Debian
sudo apt-get install ninja-build

# macOS
brew install ninja

# 或跳过 Ninja，直接在 CMakePresets.json 中将 generator 改为 "Unix Makefiles"
```

### Q: ARM 交叉编译报 "compiler not found"

确保安装了对应的交叉编译器包（参见上方"交叉编译"章节）。可以用 `which` 验证：
```bash
which aarch64-linux-gnu-g++
```

### Q: Windows 下 `.msi` 未生成

WiX Toolset 未安装或未在 PATH 中。安装后重新运行：
```powershell
dotnet tool install -g wix
```

CMake 配置输出会显示是否检测到 WiX：
```
CPack generators: ZIP;WIX    # ← WiX 可用
CPack generators: ZIP        # ← WiX 不可用，仅 ZIP
```

### Q: CTest 在 ARM 交叉编译时失败

CTest 在交叉编译目标上无法运行（二进制文件无法在宿主 CPU 上执行）。这是预期行为。如需在 ARM 上测试，可使用 QEMU 用户模式：
```bash
sudo apt-get install qemu-user
qemu-aarch64 ./build/linux-aarch64-debug/Vora examples/main.va
```
