# 运行时系统（Runtime）开发文档

## 概述

运行时系统提供 Vora 程序执行所需的基础设施：值类型系统、作用域环境、可调用对象、错误处理等。

- **相关文件**：`src/runtime/` 目录下所有文件
- **命名空间**：`vora`

## Value 类型系统

### Value 定义

```cpp
using Value = std::variant<
    std::nullptr_t,                // null
    double,                        // 数字（所有数字统一为 double）
    bool,                          // 布尔（TRUE / FALSE）
    std::string,                   // 字符串
    std::shared_ptr<Array>,        // 数组
    std::shared_ptr<Callable>,     // 可调用对象
    std::shared_ptr<ObjectInstance> // 对象实例
>;
```

### 类型对应关系

| Vora 类型 | C++ 类型 | 示例 |
|-----------|----------|------|
| `null` | `std::nullptr_t` | `null` |
| Number | `double` | `42`, `3.14` |
| Boolean | `bool` | `TRUE`, `FALSE` |
| String | `std::string` | `"hello"` |
| Array | `shared_ptr<Array>` | `[1, 2, 3]` |
| Function | `shared_ptr<Callable>` | `func f() {}` |
| Object | `shared_ptr<ObjectInstance>` | `Obj Foo() {}` |

### Array 结构

```cpp
struct Array {
    std::vector<Value> elements;  // 动态元素列表
};
```

### ObjectClass 结构

```cpp
struct ObjectClass {
    std::string className;                                  // 类名
    std::vector<std::string> params;                        // 构造函数参数名
    std::shared_ptr<BlockStmt> body;                        // 构造函数体
    std::map<std::string, std::shared_ptr<VoraFunction>> methods;  // 方法表
};
```

### ObjectInstance 结构

```cpp
struct ObjectInstance {
    std::string className;                          // 类名
    std::map<std::string, Value> properties;        // 属性表
    std::shared_ptr<ObjectClass> classDefinition;   // 所属类定义
};
```

### printValue()

```cpp
void printValue(const Value& value);
```

将 Value 输出到 `stdout`：

| 类型 | 输出格式 |
|------|----------|
| `nullptr_t` | `null` |
| `bool` | `true` / `false` |
| `double` (整数) | `42`（无小数点） |
| `double` (小数) | `3.140000` |
| `string` | 原样输出 |
| `Array` | `[1, 2, 3]`（递归，元素以 `, ` 分隔） |
| `NativeFunction` | `<native fn name>` |
| `VoraFunction` | `<fn name>` |
| `ObjectInstance` | `<ClassName object>` |

## Callable — 可调用对象体系

### 类层次

```
Callable (抽象基类)
 ├── NativeFunction    — C++ 原生函数包装
 ├── VoraFunction      — 用户定义函数（带闭包）
 ├── BoundMethodCallable  — 绑定方法（interpreter.cpp 内联定义）
 └── ObjectConstructorCallable  — 对象构造器（interpreter.cpp 内联定义）
```

### Callable 基类

```cpp
class Callable {
public:
    virtual ~Callable() = default;
    virtual Value call(Interpreter& interpreter, const std::vector<Value>& arguments) = 0;
};
```

### NativeFunction

封装 C++ 函数为 Vora 可调用对象。

```cpp
class NativeFunction : public Callable {
    using NativeFn = std::function<Value(const std::vector<Value>&)>;

    std::string name_;    // 函数名
    int arity_;           // 参数个数（-1 = 变参）
    NativeFn function_;   // 底层 C++ 函数
};
```

**构造**：
```cpp
NativeFunction("print", -1, [](const vector<Value>& args) -> Value { ... });
```

**`call()`**：直接调用 `function_(arguments)`。

### VoraFunction

用户定义的 Vora 函数，持有闭包环境。

```cpp
class VoraFunction : public Callable {
    std::string name_;
    std::vector<std::string> params_;
    std::shared_ptr<BlockStmt> body_;
    std::shared_ptr<Environment> closure_;  // 定义时捕获的环境
};
```

**`call()`**：委托给 `Interpreter::callFunction()`：
```
pushScope(closure)
定义参数
执行 body
捕获 ReturnSignal → 返回值
popScope
```

### BoundMethodCallable

在 `interpreter.cpp` 的 `evaluate()` 中内联定义。当访问对象方法时创建。

```cpp
struct BoundMethodCallable : public Callable {
    std::shared_ptr<VoraFunction> method;
    std::shared_ptr<ObjectInstance> instance;
};
```

**`call()`**：
```
pushScope(method->closure)
定义参数
定义 "this" = instance
执行方法体
捕获 ReturnSignal → 返回值
popScope
```

### ObjectConstructorCallable

在 `interpreter.cpp` 的 `execute()` 中内联定义。用于 `Obj` 声明。

```cpp
struct ObjectConstructorCallable : public Callable {
    std::shared_ptr<ObjectClass> classDef;
    std::shared_ptr<VoraFunction> constructorFn;
};
```

**`call()`**：
```
创建 ObjectInstance
设置 className 和 classDefinition
executeBlockWithThis(constructorBody, instance, params, arguments)
返回 instance
```

## Environment — 词法作用域

### 设计

Environment 形成链表结构，每个节点是一个变量哈希表，通过 parent 指针连接到外层作用域。

```
globals → env_A → env_B → env_C (current)
           ↑
         define/get/assign 沿链查找
```

### 两种父作用域持有方式

```cpp
Environment* enclosing;                    // 裸指针（不拥有）
std::shared_ptr<Environment> enclosingOwned;  // 共享指针（拥有）
```

`parent()` 方法统一返回有效的一种：
```cpp
const Environment* parent() const {
    if (enclosingOwned) return enclosingOwned.get();
    return enclosing;
}
```

### 核心方法

#### define()

```cpp
void define(const std::string& name, const Value& value);
```

在**当前帧**直接插入或覆盖。不检查是否已存在。

#### get()

```cpp
Value get(const std::string& name, const Token& token) const;
```

1. 当前帧有 → 返回
2. 有 parent → 递归 `parent->get()`
3. 都没有 → 抛出 `RuntimeError("Undefined variable: " + name)`

#### assign()

```cpp
void assign(const std::string& name, const Value& value, const Token& token);
```

1. 当前帧有 → 修改
2. 有 parent → 递归 `parent->assign()`（使用 `const_cast`）
3. 都没有 → 抛出 `RuntimeError("Undefined variable: " + name)`

#### hasLocal()

```cpp
bool hasLocal(const std::string& name) const;
```

仅检查当前帧（不递归）。

#### snapshot() — 闭包捕获

```cpp
static std::shared_ptr<Environment> snapshot(const Environment& env);
```

递归深拷贝整条作用域链。用于闭包捕获——确保函数持有定义时环境的完整快照。

```
原始: globals → env_A → env_B
快照: globals' → env_A' → env_B'   （完全独立的副本）
```

> 注意：`snapshot()` 深拷贝了 `values` 哈希表，但 `Value` 中的 `shared_ptr`（数组、对象等）是**浅拷贝**。

## RuntimeError — 运行时异常

```cpp
class RuntimeError : public std::runtime_error {
    Token token_;  // 出错位置
public:
    RuntimeError(const std::string& message, Token token);
    const Token& token() const noexcept;
    int line() const noexcept;
};
```

**捕获位置**：
- `main.cpp` 中捕获顶层脚本执行的异常
- `runREPL()` 中捕获 REPL 单行执行的异常
- 解释器内部通过 `ReturnSignal` 机制处理函数返回

**常见错误消息**：
| 消息 | 触发场景 |
|------|----------|
| `"Undefined variable: X"` | 变量未声明 |
| `"Can only call functions"` | 对非函数值进行调用 |
| `"Wrong arity: expected N but got M"` | 参数个数不匹配 |
| `"Division by zero"` | 除以零 |
| `"Indexing requires an array"` | 对非数组使用索引 |
| `"Array index must be a number"` | 索引不是数字 |
| `"Array index must be a non-negative integer"` | 索引为负数或小数 |
| `"Index out of bounds"` | 数组越界 |
| `"Invalid unary operand"` | 一元运算符操作数类型错误 |
| `"Invalid binary operands"` | 二元运算符操作数类型错误 |
| `"Can only access properties on objects"` | 对非对象访问属性 |
| `"Can only set properties on objects"` | 对非对象设置属性 |
| `"Undefined property: X"` | 属性不存在 |
| `"Cannot use 'this' outside of object method"` | 在非方法中使用 this |
| `"Can only iterate over arrays or strings"` | for-in 遍历非数组/非字符串 |
| `"Cannot use break outside of loop"` | 在循环外使用 break |
| `"Cannot use continue outside of loop"` | 在循环外使用 continue |
| `"Cannot return from top-level code"` | 顶层代码使用 return |

## RuntimeConfig — 运行时配置

```cpp
enum class RuntimeMode { Script, REPL };

struct RuntimeConfig {
    RuntimeMode mode = RuntimeMode::Script;
    bool repl() const noexcept { return mode == RuntimeMode::REPL; }
};
```

**REPL 模式特性**：表达式语句的结果自动打印。

## ObjectConstructor — 旧版构造器（已废弃）

`ObjectConstructor` 类是早期版本的对象构造器实现，已被 `interpreter.cpp` 中内联的 `ObjectConstructorCallable` 替代。代码保留但未使用。

```cpp
class ObjectConstructor : public Callable {
    std::shared_ptr<ObjectClass> classDefinition_;
public:
    ObjectConstructor(shared_ptr<ObjectClass>);
    Value call(Interpreter&, const vector<Value>&) override;
};
```

## 扩展指南

### 添加新的值类型

1. 在 `value.h` 的 `Value` variant 中添加新类型
2. 在 `printValue()` 中添加输出逻辑
3. 在 `ASTPrinter::print()` 中添加调试输出
4. 在 `Interpreter::evaluate()` 中添加相关运算逻辑
5. 在 `interpolateString()` 中添加插值转换

### 添加新的标准库函数

在 `Interpreter` 构造函数中：
```cpp
defineNative("functionName", arity,
    [](const std::vector<Value>& args) -> Value {
        // 实现
        return result;
    }
);
```

### 添加新的内建方法

对于数组、字符串等内建类型的方法，需要在 `PropertyExpr` 的求值逻辑中添加特殊分支，或者在 `ObjectClass` 中注册方法。
