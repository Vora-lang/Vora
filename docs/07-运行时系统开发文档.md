# 运行时系统（Runtime）开发文档

## 概述

运行时系统提供 Vora 程序执行所需的基础设施：值类型系统、作用域环境、可调用对象、错误处理等。

- **相关文件**：`src/runtime/` 目录下所有文件
- **命名空间**：`vora`

## Value 类型系统

### Value 定义

```cpp
using Value = std::variant<
    std::nullptr_t,                          // null
    double,                                  // float64 浮点数
    int64_t,                                 // int64 整数
    bool,                                    // 布尔（true / false）
    std::string,                             // 字符串
    GcPtr<Array>,                            // 数组（GC 管理）
    GcPtr<Dict>,                             // 字典（GC 管理）
    GcPtr<Callable>,                         // 可调用对象（GC 管理）
    GcPtr<ObjectInstance>,                   // 对象实例（GC 管理）
    GcPtr<FunctionPrototype>,                // 编译后的函数原型（VM 常量池用）
    GcPtr<ClassDefinition>                   // 编译后的类定义（统一编译期+运行时）
>;
```

### 双数值类型（int64 / float64）

Vora v0.12 引入了双数值类型系统：

| 数值类型 | C++ 类型 | 示例 | 何时产生 |
|----------|----------|------|----------|
| `int` | `int64_t` | `42`、`0xFF`、`0o77` | 无小数点、无指数的字面量 |
| `float` | `double` | `3.14`、`1e10`、`2.0` | 带小数点或指数的字面量 |

**类型提升规则**：

| 操作 | 规则 |
|------|------|
| `int ±/* int` | → `int`（无溢出检查） |
| `int ±/* float` | → `float`（int 自动提升为 float） |
| `float ±/* float` | → `float` |
| `/` 除法 | **始终** → `float64`（即使两个 int 相除） |
| 比较 `==` / `!=` | 跨类型数值相等（`42 == 42.0` → `true`） |
| 比较 `<` / `>` / `<=` / `>=` | int 自动提升为 float 再比较 |
| `type(42)` | `"int"` |
| `type(3.14)` | `"float"` |

**辅助函数**（`value.h`）：

```cpp
inline bool isNumeric(const Value& v) {
    return std::holds_alternative<double>(v) || std::holds_alternative<int64_t>(v);
}

inline double toDouble(const Value& v) {
    if (std::holds_alternative<int64_t>(v)) return static_cast<double>(std::get<int64_t>(v));
    return std::get<double>(v);
}

inline Value promoteToFloat(const Value& v) {
    if (std::holds_alternative<int64_t>(v)) return static_cast<double>(std::get<int64_t>(v));
    return v;
}
```

### ClassDefinition 结构（统一编译期+运行时）

```cpp
struct ClassDefinition : GcObject {
    // --- 编译期字段（visitObjStmt 设置）---
    std::string name;
    std::vector<std::string> parentNames;  // 多继承父类名
    std::vector<std::string> params;       // 构造函数参数
    GcPtr<FunctionPrototype> ctorProto;
    std::vector<GcPtr<FunctionPrototype>> methodProtos;

    // --- 运行时字段（OP_CLASS 设置）---
    std::map<std::string, GcPtr<VoraFunction>> methods;
    std::vector<GcPtr<ClassDefinition>> parentClasses;
    std::vector<GcPtr<ClassDefinition>> mro;  // C3 线性化
};
```

### ObjectInstance 结构

```cpp
struct ObjectInstance : GcObject {
    std::string className;                          // 类名
    std::map<std::string, Value> properties;        // 属性表
    GcPtr<ClassDefinition> classDefinition;          // 所属类定义
};
```

### FunctionPrototype 结构（VM 专用）

编译后的函数原型，存储在常量池中。`OP_CLOSURE` 指令在运行时从原型创建 `VoraFunction`。

```cpp
struct FunctionPrototype {
    std::string name;
    int arity;
    Chunk chunk;
    std::vector<UpvalueDescriptor> upvalues;  // 捕获的上值描述符
};
```

### ClassDefinition 结构（VM 专用，统一编译期+运行时）

编译后的类定义，存储在常量池中。`OP_CLASS` 指令在运行时解析父类、构建方法 `VoraFunction`、并计算 C3 MRO。

```cpp
struct ClassDefinition : GcObject {
    // 编译期
    std::string name;
    std::vector<std::string> parentNames;
    std::vector<std::string> params;
    GcPtr<FunctionPrototype> ctorProto;
    std::vector<GcPtr<FunctionPrototype>> methodProtos;

    // 运行时（OP_CLASS 设置）
    std::map<std::string, GcPtr<VoraFunction>> methods;
    std::vector<GcPtr<ClassDefinition>> parentClasses;
    std::vector<GcPtr<ClassDefinition>> mro;
};
```

## 数组内建方法

Vora v0.12 通过**属性拦截**（Property Interception）机制为数组提供 7 个内建方法。当访问 `arr.length` 或 `arr.add(item)` 时，解释器和 VM 在处理属性访问前拦截 `Array` 类型：

### 解释器拦截（`visitPropertyExpr`）

```cpp
if (std::holds_alternative<std::shared_ptr<Array>>(obj)) {
    auto arr = std::get<std::shared_ptr<Array>>(obj);
    if (expr.property == "length") return static_cast<int64_t>(arr->elements.size());
    auto method = getArrayMethod(expr.property, arr);
    if (method) return method;
    throw RuntimeError("Unknown array method: " + expr.property, expr.token);
}
```

### VM 拦截（`OP_GET_PROPERTY`）

```cpp
if (std::holds_alternative<std::shared_ptr<Array>>(obj)) {
    auto arr = std::get<std::shared_ptr<Array>>(obj);
    if (name == "length") { vm.push(static_cast<int64_t>(arr->elements.size())); break; }
    auto method = vmGetArrayMethod(name, arr);
    if (method) { vm.push(method); break; }
    throw RuntimeError("Unknown array method: " + name, errorToken);
}
```

### 方法列表

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `.length` | 0（属性） | `int64` | 元素个数，等同于 `len(arr)` |
| `.add(item)` | 1 | `null` | 尾部追加一个元素（原地修改） |
| `.pop()` | 0 | 值或 `null` | 删除并返回末尾元素；空数组返回 `null` |
| `.insert(index, item)` | 2 | `null` | 在指定位置插入；越界 → 追加到末尾；负数 → 视为 0 |
| `.remove(index)` | 1 | 值 | 删除并返回指定位置元素；越界 → `RuntimeError` |
| `.indexOf(item)` | 1 | `int64` | 首次出现的索引；未找到 → `-1` |
| `.clear()` | 0 | `null` | 清空所有元素 |

## 字符串内建方法

字符串同样通过属性拦截提供 11 个内建方法（含 `.length` 属性）：

### 解释器拦截（`visitPropertyExpr`）

```cpp
if (std::holds_alternative<std::string>(obj)) {
    auto str = std::get<std::string>(obj);
    if (expr.property == "length") return static_cast<int64_t>(str.size());
    auto method = getStringMethod(expr.property, str);
    if (method) return method;
    throw RuntimeError("Unknown string method: " + expr.property, expr.token);
}
```

### VM 拦截（`OP_GET_PROPERTY`）

```cpp
if (std::holds_alternative<std::string>(obj)) {
    const auto& str = std::get<std::string>(obj);
    if (name == "length") { vm.push(static_cast<int64_t>(str.size())); break; }
    auto method = vmGetStringMethod(name, str);
    if (method) { vm.push(method); break; }
    throw RuntimeError("Unknown string method: " + name, errorToken);
}
```

### 方法列表

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `.length` | 0（属性） | `int64` | 字符数，等同于 `len(str)` |
| `.substring(start, end?)` | 1-2 | `string` | 子串 [start, end)；负数从末尾计；省略 end 到末尾 |
| `.include(sub)` | 1 | `bool` | 是否包含子串；空串始终为 `true` |
| `.startsWith(prefix)` | 1 | `bool` | 是否以 prefix 开头 |
| `.endsWith(suffix)` | 1 | `bool` | 是否以 suffix 结尾 |
| `.upper()` | 0 | `string` | 转为大写（返回新字符串） |
| `.lower()` | 0 | `string` | 转为小写（返回新字符串） |
| `.trim()` | 0 | `string` | 去除首尾空白字符 |
| `.replace(old, new)` | 2 | `string` | 替换**首次**出现的 old → new |
| `.replaceAll(old, new)` | 2 | `string` | 替换**所有**出现的 old → new |
| `.split(delim)` | 1 | `Array` | 按分隔符切分为字符串数组；`""` → 逐字符 |

### 方法实现

所有方法实现为 `NativeFunction`：
- **数组方法**：通过 `shared_ptr<Array>` 捕获数组引用，原地修改
- **字符串方法**：通过值捕获 `std::string`，返回新字符串（字符串不可变）

两套后端（解释器和 VM）各有一套独立的辅助函数（`getArrayMethod`/`vmGetArrayMethod`、`getStringMethod`/`vmGetStringMethod`），但业务逻辑相同。

## Callable — 可调用对象体系

### 类层次

```
Callable (抽象基类，继承 GcObject)
 ├── NativeFunction    — C++ 原生函数包装
 ├── VoraFunction      — 用户定义函数（VM 编译模式）
 ├── BoundMethod       — 方法绑定到实例
 └── ClassConstructor  — 类构造器 + MRO 构造链
```

### Callable 基类

```cpp
class Callable : public GcObject {
public:
    virtual ~Callable() = default;
    virtual Value call(const std::vector<Value>& arguments) = 0;
    virtual GcPtr<ClassDefinition> getClassDef() const { return GcPtr<ClassDefinition>(); }
};
```

### NativeFunction

封装 C++ 函数为 Vora 可调用对象。

```cpp
class NativeFunction : public Callable {
    using NativeFn = std::function<Value(const std::vector<Value>&)>;

    std::string name_;    // 函数名
    int arity_;           // 参数个数（-1 = 变参）
    NativeFn function_;   // 底层 C++ 函数
};
```

**构造**：
```cpp
NativeFunction("print", -1, [](const vector<Value>& args) -> Value { ... });
```

**`call()`**：直接调用 `function_(arguments)`。

### VoraFunction

用户定义的 Vora 函数（VM 编译模式）。

```cpp
class VoraFunction : public Callable {
    std::string name_;
    int arity_ = 0;
    int requiredArity_ = 0;  // 不含默认值的参数个数
    const FunctionPrototype* prototype_;
    std::vector<std::shared_ptr<Value>> upvalues;  // 上值（VM 闭包用）
};
```

**`call()`**：委托给 `VM::callVoraFunction()` 创建新帧并执行字节码。

### BoundMethod

将方法绑定到特定实例。VM 在 `OP_GET_PROPERTY` 后创建，将 `this` 绑定到方法调用。

```cpp
class BoundMethod : public Callable {
    GcPtr<VoraFunction> method_;
    GcPtr<ObjectInstance> instance_;
};
```

### ClassConstructor

类实例工厂。负责创建 `ObjectInstance` 并按 C3 MRO 逆序执行构造链（根类最先）。

```cpp
class ClassConstructor : public Callable {
    GcPtr<ClassDefinition> classDef_;
};
```

**`call()`**：
```
创建 ObjectInstance（设置 className + classDefinition）
按 MRO 逆序执行每个父类的构造函数体
执行自身的构造函数体
返回 instance
```

## Environment — 词法作用域

### 设计

Environment 形成链表结构，每个节点是一个变量哈希表，通过 `enclosing_`（shared_ptr）连接到外层作用域。v0.18 起统一使用 `shared_ptr`，消除了裸指针和 `const_cast` 问题。

```
globals → env_A → env_B → env_C (current)
           ↑
         define/get/assign 沿链查找
```

### 核心方法

#### define()

```cpp
void define(const std::string& name, const Value& value);
```

在**当前帧**直接插入或覆盖。不检查是否已存在。

#### get()

```cpp
Value get(const std::string& name, const Token& token) const;
```

1. 当前帧有 → 返回
2. 有 parent → 递归 `parent->get()`
3. 都没有 → 抛出 `RuntimeError("Undefined variable: " + name)`

#### assign()

```cpp
void assign(const std::string& name, const Value& value, const Token& token);
```

1. 当前帧有 → 修改
2. 有 parent → 递归 `parent->assign()`
3. 都没有 → 抛出 `RuntimeError("Undefined variable: " + name)`

#### hasLocal()

```cpp
bool hasLocal(const std::string& name) const;
```

仅检查当前帧（不递归）。

#### snapshot() — 闭包捕获

```cpp
static std::shared_ptr<Environment> snapshot(const Environment& env);
```

递归深拷贝整条作用域链。用于闭包捕获——确保函数持有定义时环境的完整快照。

```
原始: globals → env_A → env_B
快照: globals' → env_A' → env_B'   （完全独立的副本）
```

> 注意：`snapshot()` 深拷贝了 `values` 哈希表，但 `Value` 中的 `shared_ptr`（数组、对象等）是**浅拷贝**。

## RuntimeError — 运行时异常

```cpp
class RuntimeError : public std::runtime_error {
    Token token_;  // 出错位置
public:
    RuntimeError(const std::string& message, Token token);
    const Token& token() const noexcept;
    int line() const noexcept;
};
```

**捕获位置**：
- `main.cpp` 中捕获顶层脚本执行的异常
- `runREPL()` 中捕获 REPL 单行执行的异常
- 解释器内部通过 `ReturnSignal` 机制处理函数返回
- VM 内部通过 `OP_THROW` + `catchHandlers` 栈处理异常

**常见错误消息**：
| 消息 | 触发场景 |
|------|----------|
| `"Undefined variable: X"` | 变量未声明 |
| `"Can only call functions"` | 对非函数值进行调用 |
| `"Wrong arity: expected N but got M"` | 参数个数不匹配 |
| `"Division by zero"` | 除以零 |
| `"Indexing requires an array"` | 对非数组/非字符串使用索引 |
| `"Array index must be a number"` | 索引不是数字 |
| `"Array index must be a non-negative integer"` | 索引为负数或小数 |
| `"Index out of bounds"` | 数组越界 |
| `"Invalid unary operand"` | 一元运算符操作数类型错误 |
| `"Invalid binary operands"` | 二元运算符操作数类型错误 |
| `"Can only access properties on objects"` | 对非对象/非数组/非字符串访问属性 |
| `"Can only set properties on objects"` | 对非对象设置属性 |
| `"Undefined property: X"` | 属性不存在 |
| `"Unknown array method: X"` | 数组方法不存在 |
| `"Unknown string method: X"` | 字符串方法不存在 |
| `"Cannot use 'this' outside of object method"` | 在非方法中使用 this |
| `"Can only iterate over arrays or strings"` | for-in 遍历非数组/非字符串 |
| `"Cannot use break outside of loop"` | 在循环外使用 break |
| `"Cannot use continue outside of loop"` | 在循环外使用 continue |
| `"Cannot return from top-level code"` | 顶层代码使用 return |

struct RuntimeConfig {

## VM 内建函数

v0.13 起 VM 内建函数在 `src/runtime/builtins.cpp` 中集中定义（`registerBuiltins(VM&)` 函数）。当前内建函数（12 个）：

| 函数 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `print(...)` | 变参 | `null` | 打印所有参数（空格分隔），自动换行 |
| `input(prompt?)` | 0-1 | `string` | 读取一行输入 |
| `assert(cond, msg?)` | 1-2 | `null` | 断言，失败抛出 RuntimeError |
| `type(value)` | 1 | `string` | 返回类型名：`"int"`, `"float"`, `"string"`, `"boolean"`, `"array"`, `"dict"`, `"function"`, `"object"`, `"class"`, `"null"` |
| `int(value)` | 1 | `int64` | 字符串→整数转换 |
| `float(value)` | 1 | `float64` | 字符串→浮点数转换；整数保留为 float |
| `len(value)` | 1 | `int64` | 数组元素数 / 字符串字符数 / 字典键数 |
| `range(stop)` / `range(start, stop)` / `range(start, stop, step)` | 1-3 | `Array` | 生成数字数组 |
| `clock()` | 0 | `float64` | 返回当前时间（秒） |
| `bin(n)` | 1 | `string` | 整数→二进制字符串 `"0b..."` |
| `oct(n)` | 1 | `string` | 整数→八进制字符串 `"0o..."` |
| `hex(n)` | 1 | `string` | 整数→十六进制字符串 `"0x..."` |

其中 `len()` 和 `type()` 是 v0.12 新增的内建函数。

## 扩展指南

### 添加新的值类型

1. 在 `value.h` 的 `Value` variant 中添加新类型
2. 在 `printValue()` 和 `valueToString()` 中添加输出逻辑
3. 在 `ASTPrinter::print()` 中添加调试输出
4. 在解释器 `evaluate()` 中添加相关运算逻辑
5. 在 VM `run()` 中添加相关运算逻辑
6. 在 `interpolateString()` 中添加插值转换

### 添加新的标准库函数

**解释器**：在 `Interpreter` 构造函数中：
```cpp
defineNative("functionName", arity,
    [](const std::vector<Value>& args) -> Value {
        // 实现
        return result;
    }
);
```

**VM**：在 `main.cpp` 中：
```cpp
vm.defineNative("functionName", arity,
    [](const std::vector<Value>& args) -> Value {
        // 实现
        return result;
    }
);
```

### 添加新的内建方法

对于数组、字符串等内建类型的方法，使用**属性拦截**模式：

1. 在解释器的 `visitPropertyExpr()` 中添加类型检查和分发逻辑
2. 在 VM 的 `OP_GET_PROPERTY` 处理器中添加类型检查和分发逻辑
3. 编写 `getXxxMethod()` 辅助函数，返回 `shared_ptr<NativeFunction>`
4. 属性获取器（如 `.length`）直接返回 `Value`，不走 NativeFunction
